단골 기안을 작업으로 저장하면 매핑·표시형으로 데이터가 자동으로 채워집니다. 일회성 기안은 왼쪽 레일의 「빠른 기안」으로.

주석 — 결정 대응표

  1. 1화면명 「작업」·부팅 랜딩. 부제가 멘탈 모델을 진술. KPI·이어서 실행·태그 바는 부재가 곧 설계(⑥·⑦·⑧).
  2. 2화면급 툴바 — 정렬 3종(기본 이름순·선택 영속)과 이름 필터가 두 열 공통. 스크롤과 무관하게 상시 노출.
  3. 3이름순 기본이 만드는 변형 군집 인접(입찰공고-2천만미만/-5천만미만/-긴급/-재공고) — 금액구간 선택은 데이터가 결정.
  4. 4v5 일화 강등 제안: 「마지막 실행: …」 텍스트를 행 면에서 제거 — 회수의 마지막 수는 데이터 대조(③)라 일화가 일상 루프에 하중이 없고, CC 행 문법도 이름-단독. 일화 경로는 호버 툴팁으로 강등(이 시안에서 실제로 행에 마우스를 올리면 브라우저 툴팁으로 뜸) — 영구 소음 0, 필요 시 접근 가능. 시간 정보가 필요하면 정렬 「최근 실행순」이 대신함. 확정
  5. 5v4 행 클릭 = 주동사(CC 이식): 행 자체를 클릭하면 출력별 세션 진입(왼쪽=실행, 오른쪽=기안 채우기 — 생성은 강제확인 게이트 뒤라 비파괴 내비). 동사 버튼 소멸, ⋯는 호버·포커스 시에만 노출(이 행이 호버 상태의 정적 데모 — 실제로도 마우스를 올리면 나타남).
  6. 6경보 행: 끊긴 작업은 행 전체가 경보 틴트, 「템플릿 다시 연결…」이 상시 노출 — 경보 어포던스는 호버 뒤에 숨지 않는다. 행 클릭도 실행이 아니라 재연결로. 화면급 손상 경보는 현행 유지(조건부).
  7. 7개수 표기 규칙 확정: 펼친 그룹·구획 머리의 개수는 전부 제거(아래 행이 보이는데 세는 것 = KPI와 같은 과시 함정). 접힌 그룹의 "(N)"만 유지 — 보이지 않는 내용물의 유일한 표지(F8 접힘 문법·confirm-or-alarm). 과시/표지의 경계 = 접힘 여부. 확정
  8. 8창작 어포던스 — 열마다 ＋(매체 맥락 상속). 미확정 3
  9. 9v2 스크롤 처리: 카드→행(~34px, 2.5배 밀도) + 화면 전체 스크롤 제거 — 각 열의 목록만 독립 스크롤(열 머리·툴바 고정). N=13이 한 파에 다 보이고, N=40이어도 ~1.5파 + 필터 지름길.
  10. 10v3 간단 그룹화: 작업당 선택적 그룹 하나(단일 소속) — 다축 태그(⑥ 제거)의 부활이 아님: 축도 DSL도 렌즈도 없음. 무그룹이면 v2와 완전히 동일(퇴화-코퍼스 불변식). 정렬은 그룹 내 적용, 그룹 배열은 이름순 고정·그룹 우선(탐색기 관례). 확정
  11. 11접힘 — 헤더 클릭으로 그룹 접힘, 「이름 (N)」이 곧 표지(F8 접힘 문법: 사라지는 게 아니라 접힘). 안 쓰는 계열을 접어 단골만 한 파에 남기는 게 이 기능의 실효.
  12. 12v4 메뉴 문법(CC 이식): 편집 / 복제 / ─ / 그룹으로 이동 ▸(서브메뉴: 기존 그룹 목록 + 현재 소속 ✓ + 새 그룹… + 그룹 해제 — 모달 왕복 제거) / ─ / 삭제(위험 동사는 빨강·맨 아래 격리 + 확인 모달). 구현은 modal.js 계열, 네이티브 금지(F7 교훈).
  13. 13v4 「그룹 없음」 명명 섹션(CC 이식): 그룹이 하나라도 있으면 무그룹 작업도 라벨 있는 접이식 구획에 — 모호한 중간 상태 제거. 그룹이 0개면 헤더 자체가 없는 평면(불변식 유지).
  14. 14v6 들여쓰기 강화(CC 수준 26px) + 그룹 간 여백 > 행 간 여백(헤더 상단 12px) — 그룹 위계가 들여쓰기(수평)와 여백(수직) 두 축으로 읽힘. 그룹 0개 평면에선 들여쓰기 없음(불변식).
  15. 15v5.1 정렬 규칙(수정): 그룹 배열은 안정 — 이름순 고정, 「그룹 없음」 마지막. 정렬 키는 그룹 내부에만 적용. 앞판(대표값으로 그룹도 재배열)은 폐기 — CC가 프로젝트 그룹에서 활동순 자동 배열을 썼다가 "그룹이 뛰어다닌다"는 사용자 불만(이슈 #70104)으로 안정 배열 요구를 받은 실전 반증 + 그룹 위치의 공간 기억 보존. 그룹은 소수(≤8)라 최근 항목 찾기에 손해 미미.
  16. 16v6 열 폭 상한: 일화 텍스트 소멸로 행=이름-단독이 되자 사이드바 유래의 과폭이 드러남 → 열 폭을 내용에 맞게 상한(400/360px). 남는 폭은 당장 정직한 여백으로 두고, 구조적 세입자 후보 = 변형 B의 master-detail(행 클릭 → 오른쪽 실행 세션). 채택 시 「작업」이 일상 루프 전체의 단일 표면이 되고 레일 「실행」이 잉여화 — R-flow 라운드 소관으로 인계, 여기선 미결정.
  17. 17v6 그룹 접힘 화살표(CC 문법): 이름 오른쪽·평시 숨김·호버 노출(헤더에 마우스를 올리면 나타남). 단 접힌 그룹은 화살표 상시 노출 — 접힘 상태 표지는 숨기지 않는다(confirm-or-alarm의 시각판).
  18. B0초기 상태(선택 전): 변형 B는 v6과 경쟁하는 별개 대안 — 두 열 상태가 존재하지 않고 txt 구획은 처음부터 왼쪽 목록의 두 번째 구획. 오른쪽은 빈 패널로 시작, 클릭은 패널 내용만 채움(레이아웃 불변·애니메이션 불요). 구획 헤더의 색 막대(파랑/초록)가 열 틴트를 승계해 맥락 분리를 시각으로 유지. 접힘 상태는 사용자 소유·영속(자동 접힘 없음 — 철회 합의).
  19. B1변형 B 목록 단일화: master-detail에선 두 열이 한 목록의 두 구획(문서/기안 섹션)으로 접힘 — CC 사이드바와 같은 형상. 짝-날엔 같은 목록 안에서 구획만 오르내림.
  20. B2선택 상태: 클릭-고정(호버 아님) — 선택 행이 틴트+좌측 막대로 상주 표시. 실행 패널의 주인이 항상 목록에 보임.
  21. B3in-pane 실행 세션: 행 클릭 → 이 자리에 세션이 열림, 화면 전환 0. "누르면 넘어가버림"의 소멸이 master-detail의 본뜻. 세션 내부(데이터→행→게이트→생성)는 R-flow에서 확정.
  22. B4세션 가드(R-flow 쟁점): 입력 진행 중 다른 행을 클릭하면 조용히 버리지 않고 확인(editJob 미저장 가드와 대칭). 강제확인 게이트는 그대로 상속.
  23. B5파급: 채택 시 레일 「실행」 잉여화(회색 표시는 시안 표기일 뿐) · 실행 화면의 작업 드롭다운 소멸 — 앞서 세운 경계 표지가 실제로 발효되는 지점.
  24. B6txt 세션 = 같은 4존 패널 문법(헤더·데이터·본문·완료), 내부만 출력별: 데이터는 한 행(기안은 단건), 본문은 매핑 선채움+라이브 잔여+미리보기, 완료는 「복사」. 행위 층 분리가 패널 내부에서 실현됨(원칙 ③).
  25. B7표시형 선채움 실물: "38,500,000원" — cli.py:169이 증명한 코어 경로(매핑+format_engine)의 GUI 착지. F36 서식 갭이 소멸하는 그 지점. 미채움 필드는 미리보기에 {{담당자}}로 정직하게 노출.
  26. B8빈칸 게이트: hwpx의 강제확인 게이트 대신 기안은 미채움 경보 — 산출물 무게(법적 문서 파일 vs 클립보드)에 비례한 게이트 무게. 조용한 빈칸 금지(confirm-or-alarm).
  27. B9복사 = 완료 이벤트: 여기서 last_run_at 스탬프 — 「최근 실행순」 정렬·일화 툴팁의 txt 쪽 의미론이 이걸로 확정.
  28. B11접힘-펼침 정책: ①마지막 상태 영속(Python 설정 — webview 저장소 금지, #74 전례) ②무상태 기본=전부 펼침, 새 그룹=펼침 ③필터는 접힘 관통(매치 그룹 임시 펼침, 해제 시 영속 상태 복원) ④경보는 접힘 관통 — 자동 펼침 대신 접힌 헤더에 ⚠ 표지(문제는 접혀도 시끄럽게) ⑤선택 행의 그룹을 접어도 선택 유지.
  29. B10규모 검증(N=50): 접힘 정상상태(그룹 헤더 8 + 펼친 단골 1그룹 + 구획 헤더 2 ≈ 한 파)가 실사용 수렴점 — 접힘 영속이 이 수렴을 만든다. 최악(전부 펼침 ≈ 2.5파)도 목록 독립 스크롤 + 이름순 위치 고정 + 필터로 감당. 안전판 둘 예비: 구획(문서/기안) 자체도 접힘 가능 · 그룹 없음 비대 시 그룹화가 자연 유도됨(헤더의 (10)이 스스로 신호).
