이 시안이 보여주는 것

B안 v2 — 캔버스의 핀과 대응

1

매체 = 구조, 그룹 = 그 안에서

HWPX/TXT는 동사를 가르는 경성 축이라 구획으로 나눕니다(현실이 자명 — 배울 규칙 아님). 그 안에서만 작업과 같은 그룹+접힘. 관리 로직은 하나.

2

「그룹 없음」이 받아냄

방금 가져왔거나 탐색기로 떨군 템플릿은 조용히 사라지지 않고 각 매체의 여기로. 「＋ 그룹 지정」으로 승격 — 이 동사는 헤더가 그룹을 못 알려주는 여기서만 노출.

3

가져오기 = 루트로 복사

파일을 라이브러리 단일 루트로 복사(제자리 등록 아님). 루트 파편화 차단 — "습관 고정" 원칙. 복사본은 처음엔 「그룹 없음」.

4

에디터 1단계 = 고르기 전용 (HWPX만)

마법사가 .hwpx를 만들므로 HWPX 서식만 자동 필터 — 소비 표면은 늘 단일 매체. 같은 그룹 목록을 선택 전용으로 재사용. 옛 파일버튼 4개는 「가져오기…」 하나로 붕괴(F16).

･

그룹 라벨은 카드에 반복 안 함

이미 그룹 헤더 아래 있으니 칩은 잉여. 그룹 이동은 호버 ⋮ 메뉴로(작업 모델 동형). 라벨이 아니라 동사만 남김.

재기록 대상 — 1부 결정 ③(서브폴더=폴더트리 그룹, 앱이 미러링) 폐기·교체. 근거: 작업 group+접힘 모델 구현됨 + 멘탈 모델 불증가 우선. 단 매체 축은 템플릿 고유(작업엔 없음)라 구조로 보존.

실제 앱 토큰(royal blue #2f5fbf · 맑은고딕 스택 · 간격·모서리 스케일)으로 렌더. 상호작용은 정적 시안 — 그룹 접힘/선택은 최종 형상 감을 주기 위한 표시입니다. 라벨(점검 등)은 R-copy 방향으로 손봤으나 확정 아님. 이 문서는 토의용 시안이며, 합의 시 docs/r-info-mockups/로 박제됩니다.
