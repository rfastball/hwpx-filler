이 시안이 주장하는 것

확정매핑 UI 는 하나

두 표면은 둘 다 열 이름으로 결속하고 같은 제안 임계(0.6)를 쓴다 — 값으로 결속하는 구현은 애초에 없었다. 실제로 갈렸던 건 지속성뿐이라, 한 표에 스위치를 단다.

전환기를 눌러 보면 저장 모드에서만 유형·확정 열이 붙는다.

확정데이터 존은 승계다, 재구현이 아니다

자모 부분일치 실시간 검색·엑셀식 열 필터 좁힘·필터 칩·관통 스트립· 「직전 필터 재적용」은 hwpx 트랙이 블록 4에서 세운 자산이고, txt 큐가 이미 같은 DataZoneMixin 을 쓴다(작업 화면과 단일 출처).

자모 검색은 초성 검색이 아니라 부분일치다 — 「반」이 「바나나」에 걸리고, IME 로 조립하는 중간 상태(「통학차ㄹ」)가 이미 맞는다. 「ㅌㅎㅊㄹ」 같은 초성 나열은 연속 자모 부분열이 아니라 안 걸린다(core/jamo.py 선언 경계).

합병해도 그대로 산다 — 여기서 다시 만들면 안 된다. 관통 스트립은 특히 「필터 밖 선택을 숨기지 않는다」는 계약이라 조용히 떨어뜨리면 큐가 거짓말을 한다.

확정단건 = 큐 길이 1 · 데이터 없음도 길이 1

선택 12행이면 카드 12장, 1행이면 1장. 화면은 바뀌지 않는다 — 같은 기안 작업이 오늘 1건 내일 30건이어도 같은 얼굴이어야 한다.

데이터를 아예 안 물려도 같은 규칙이다(결정 14) — 큐가 길이 1(가상 행)로 퇴화하고 큐 장치가 통째로 숨는다. 세 번째 전환 버튼이 그 상태다: 빠른 기안의 최단 경로 (붙여넣고 직접 채워 복사). 데이터가 필수인 것처럼 말하지 않는 게 ①존 빈 상태의 의무다.

확정직접 입력을 저장하면 상수가 된다

데이터 없이 만든 기안을 저장하면 매핑이 전부 수기 값이다. 그대로 굳히면 다음에 지난 건의 값이 그대로 나온다 — 조용한 오류.

기존 문법으로 풀린다: 수기 값은 상수(type="const")로 이관되고, 승격은 전 행 미확정 착지라 확정하며 「늘 같은 값」과 「이번 건 값」을 가른다. 「매번 입력」이라는 새 유형은 없다 — 값이 빈 상수가 곧 그것이고, 안 채우면 빈칸 게이트에 걸린다(「비워둠」은 반대로 게이트에서 빠진다).

확정매체는 제목이 말한다

구획 헤더가 하던 일을 화면 제목이 승계 — 「기안」 + TXT 배지 + 초록 틴트. 「작업」은 파랑을 그대로 쓴다.

확정승격은 제자리

「기안으로 저장」의 목적지가 같은 화면 좌 목록이라, 저장해도 하던 일이 끊기지 않는다. 휘발 행이 그대로 위 목록으로 올라간다.

확정원문은 같은 칸의 두 번째 모습

칸을 쪼개거나 탭을 만들지 않고, ③에서 「채운 모습」 ↔ 「원문」을 전환한다 — 산출물이 평문이라 둘은 같은 물건이다. 휘발 세션의 입력구가 여기고, 타이핑하면 ② 표가 실시간 재구성된다(결정 34).

편집 가능 여부가 네 번째 지속성 스위치다: 저장된 기안은 읽기 전용(정의가 조용히 갈라지지 않게), 손보려면 「사본으로 편집」이 휘발 세션으로 갈라 준다 — 값·데이터는 승계.

확정진행 중 원문 편집은 한 번만 묻는다

이미 복사한 건은 되돌릴 수 없다 — 클립보드를 거쳐 내부 시스템으로 갔다. 그러니 「진행 초기화」는 거짓말이고, 카드별로 어느 문안이었는지 기록하는 건 과하다.

편집을 시작할 때 한 번 사실을 진술하고(진행이 있을 때만), 확인하면 진행 상태를 그대로 둔 채 아무것도 추가로 기록하지 않는다. 원문 뷰에서 타이핑해 보라.

확정미루기 사망 · 자유 이동이 대체

결정 19가 미루기를 「막힌 카드의 유일한 탈출구」라 부른 전제는 작업점 고정이었다. 진행 색인 점 클릭과 ◀▶ 가 자유 이동을 주면 그 전제가 깨진다 — 막히면 다음 점을 누르면 된다.

진행 색인·멱등 재복사·완주 판정은 세션 휘발이라 그대로 산다. 복사 플래그는 통상대로 일방향(되돌리는 API 없음).

확정「비워둠」은 확인에서 빠진다

「비고」 행을 보라 — 열을 안 겨누고 비운다고 선언했다. 카드에는 데이터 빈칸과 똑같이 〈빈 값〉으로 그린다(결과가 같으므로 표지를 넷으로 늘리지 않는다).

다만 복사 전 확인에서는 빠진다 — 확인한 것을 다시 묻지 않는다. 데이터가 빈 것은 사람의 선언이 아니라 그 행의 사실이라 확인에 남는다.

유보토큰이 많아지면

「토큰 40개」는 hwpx 의 성질(누름틀)이지 txt 기안문의 실측된 성질이 아니다. 근거 없는 대비 설계는 짓지 않는다 — 재개 조건 = 실제 토큰 수 실측.

넘으면 「미확정만 보기」·접힘 문법을 같은 표의 헤더에 얹는다. 별도 UI 는 아니다.
