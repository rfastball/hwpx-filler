창 1440×820 시뮬레이션 — 패널 안을 실제로 스크롤해 보세요.

A안 — 평면 2열 병치 (②|③ 나란히, ③ sticky)

  • 루프가 한 층 — 타이핑→반영→복사→다음이 모드 전환 없이 흐른다(§12 병렬 패널=흐름 유지).
  • ② 캡 제거 — 중첩 스크롤 폐지, 표가 길어져도 ③은 sticky로 제자리(파생 이슈 4 해소).
  • 창폭 확대 전제 — 1180 토글에서 보듯 좁으면 현행 적층으로 폴백(=문제 그대로).
  • 스트레스 22토큰에서도 성립하나, ①존은 여전히 세로 예산을 먹는다(→ 접힘은 후속 판단).

B안 — z축 전용 편집면 (메인=요약+미리보기, 오버레이=맞추기)

  • 창폭 불문 — 1180 그대로도 성립. 메인 층이 한 화면에 다 들어온다(③이 처음으로 주인공).
  • 편집면 시야 최대 — 오버레이가 창 전체를 쓰므로 22토큰 스트레스에서 가장 쾌적.
  • 모드 비용 — 맞추기가 주 작업인 세션에선 열고-닫기가 반복 마찰(§12 모달=흐름 차단). 미결 발견→수정 왕복마다 2클릭.
  • 층 계약 비용 — 해제·포커스 귀환·Escape(#210·#211 부류) 가드가 스코프에 추가된다.

C안 (개정) — 2열 병치 + 넉넉 캡 + ⤢ 전용 편집면 (각 불만에 각 도구)

  • 병치가 불만 2(편집하며-보기)의 해법, 캡+⤢가 불만 4(대량 스트레스)의 해법, 창폭 확대가 병치의 전제 — 캡은 병치를 대체하지 못한다(캡+세로 적층이면 불만 2 미해결).
  • 캡은 평시 무발동 크기(기본 8토큰이 스크롤 없이 다 보이는 ~300px) — 발동했을 때만 존재가 드러난다.
  • 캡 발동 = 정직한 잘림 표지 — 「전체 N행 — 펼쳐서 맞추기 ⤢」. 조용한 잘림은 두지 않는다(confirm-or-alarm).
  • 표면 이원의 비용 — 같은 ②·③이 두 곳에 뜬다. 구현은 DOM 이사(단일 실체)로 상태 발산을 차단(복제 렌더 금지 — #138 F8·F11 교훈).

작업(HWPX) 대칭 적용 — 같은 병리, 같은 처방 (화면 토글로 확인)

  • 루프의 대칭: 기안=타이핑→미리보기 반영 / 작업=행 선택→거울 반영. 현행은 둘 다 세로 적층이라 조작면과 반영면이 동시에 안 보인다 — 작업은 미리보기가 없어 문제가 늦게 드러났을 뿐.
  • duo = 데이터(고르는 면) | 채울 값=거울(비치는 면, sticky). 거울은 현행 무캡(40필드면 무한정) → 캡 420px + 잘림 표지 + 「펼쳐서 확인 ⤢」.
  • 작업의 전용 면은 「편집」이 아니라 「확인 의식」의 면 — 빈 값 클릭 확인과 재진술 게이트가 같은 면에서 오간다(생성 게이트의 정직성 유지).
  • 구현 공유: duo CSS·capstrip·시트 골격·이사 프로토콜 전부 클래스 재사용 — 기안 착지 후 작업 편입 비용은 작다.

①데이터에도 같은 펼치기 — 열 정책: 「선별」이 아니라 「고정」 (C안·작업 화면의 ⤢로 확인)

  • 실코드 확인: 인라인 표는 이미 전 열을 싣는다(data_zone.py — columns 전체). 자동 선별은 선두 「문서」 열의 식별 요약(identity_summary, 결정 37)이 담당 — 좁은 상자+가로 스크롤이 사실상의 열 잘림이고, 식별 요약은 그 보상 압축이다.
  • 펼친 면 = 전체 진실: 전 열 · 원본 순서(엑셀 대조 정신 모델 보존). 열 자동 선별을 펼친 면에 가져가지 않는다 — 그것은 좁은 면의 압축 장치라, 넓은 면에 이식하면 목적 없이 정보만 숨긴다.
  • 식별성 담보는 선별 대신 고정: 선두 식별 열(체크+식별 요약)을 왼쪽 sticky, 머리행을 위 sticky — 오른쪽 끝 열을 스캔해도 어느 행인지 잃지 않는다. 식별 요약의 역할이 「대표 열 노릇」에서 「스캔 앵커」로 바뀐다.
  • 필터·선택은 같은 정의가 따라온다: 검색·칩·전체 선택·필터 밖 선택 스트립까지 이사 — 면이 달라져도 판정은 하나(FilterModel 단일 출처, 「판정은 Python」 유지).
  • 부담 통제: ⤢는 존마다 다른 발명이 아니라 「캡 있는 그리드 존의 공통 출구」 한 문법 — 같은 시트 기계(modal.js)·같은 이사 프로토콜·같은 표지 문안을 존별 내용으로 여는 것.
