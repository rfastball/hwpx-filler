# UI 용어·단위·배치 규약 (정준 용어표)

이 문서는 **앱 B(생성기)의 사용자-가시 문구·수량 단위·폼 배치**의 단일 정준 규약이다.
RC-26(라운드 1, U12 `b733c76` 착지)이 확정한 용어표를 **계승**하고, UI 디자인 라운드
V15(UD-18·20·34·41)가 그 잔여 치환·경계 정의·단위 규약·배치 규약을 못박아 완결한다.

**원칙**("1개념 1이름"): 같은 사용자 개념은 전 표면에서 **한 이름**으로 부른다 —
정상 경로 라벨뿐 아니라 **오류 다이얼로그·모달 제목·지시문·로그·카드 메타**까지.
**코드 식별자(내부 심볼)는 유지**한다: `fieldize`(CLI 명령·`apply_fieldize`),
`CompileState`, `blank`, `base_*`, `missing`/`empty_valued` 등은 그대로 두고
**사용자 문구만** 정준 용어로 렌더한다.

## 1. 정준 용어표 (RC-26 계승 + V15)

| 사용자 개념 | 정준 용어 | 폐기된 옛/내부 어휘 |
|---|---|---|
| 재사용 매핑 산출물(`MappingProfile`, `*.mapping.json`) | **매핑 프로파일** | 어휘 워크벤치·공유 베이스(매핑)·공유 어휘·베이스 |
| lint 통제 사전 | **어휘** | (‘어휘’는 이 개념에만) |
| 평문 토큰 → 누름틀 변환(사용자 문구) | **누름틀 변환** | 컴파일·미컴파일·fieldize(영문 노출) |
| 판본 쌍 | **구판 / 신판** | 이전 판본 / 새 판본 |
| 데이터 항목(매핑 대상 소스 열 — 필드당 단일선택) | **데이터 항목** | 소스(지시문에서 열을 가리킬 때) |
| 데이터 소스 종류(스텝2 라디오) | **데이터 소스** | (‘소스’ 단독 사용 금지 — 열과 혼동) |
| 재사용하도록 등록한 데이터 참조 1건 | **데이터셋** | 풀 항목 |
| 등록한 데이터 참조를 관리하는 화면 | **데이터 관리** | 데이터 풀 |
| 선택 버튼·대화상자에서의 등록 항목 집합 | **등록 데이터** | 데이터 풀 |
| 매핑 값 유형(누름틀당 하나) | **텍스트 / 날짜 / 금액 / 고정값** | 변환·결합(join·N→1)·구분자(sep)·다중소스 |
| 소스 무관 리터럴 값 유형 | **고정값** | 상수(const 노출) |
| 날짜 유형 표시형 프리셋 | **시각 / 날짜+시각**(외 한글·ISO 등) | 일시(유형명으로) |
| 통합문서 안의 데이터 표 1장(T2) | **시트** | 워크시트·탭·sheet(영문 노출) |

**엄격 1:1 매핑 모델**(코어 미러): **누름틀 하나 = 데이터 항목 하나 + 값 유형 하나**. 한 템플릿
필드는 정확히 한 데이터 항목(`source` 단일선택)을 한 값 유형으로 서식한다 — **N→1 결합·구분자
(sep)·다중소스는 모델에서 제거**됐다. 값 유형은 **텍스트 / 날짜 / 금액 / 고정값** 넷뿐이며,
`고정값`(옛 ‘상수’)은 소스와 무관한 리터럴이다. 날짜+시각처럼 예전에 두 키를 합치던 자리는
소스가 이미 합쳐진 단일 키를 주거나(권장) 각각 별도 필드로 매핑한다.

**액션당 라벨 1개**(UD-18 A1):

| 액션 | 정준 라벨(버튼·창 제목 동일) |
|---|---|
| 새 HWPX 문서 작업 생성 | **＋ 새 문서 작업** (헤더·빈 상태 동일) |
| 저장된 작업 편집 | **작업 편집** (카드 버튼 ↔ 창 제목 동일) |
| txt 기안 텍스트 저장 | **텍스트 파일로 저장** (버튼 ↔ 다이얼로그 제목 동일) |

## 2. 상태 어휘 3정의 경계 (UD-20 — ADR-B/E 정합)

같은 화면·트랙 교차에서 “같은 상태 2이름”·“1단어 2개념”을 없애기 위해 상태축을 3정의로
고정한다:

| 용어 | 정의 | 표면 |
|---|---|---|
| **미입력** | 출력값이 비어 채워지지 않음 — **ack(확인) 대상** | 실행 화면 배지·게이트·사전검증 경고 |
| **빈 값** | 원천 데이터의 값이 비어 있음(항목은 존재) | 실행·txt·미리보기 |
| **(비움)** | 사람이 **의도적으로** 채우지 않겠다고 선언 | 매핑 확정·미리보기 |
| **항목 없음** | 데이터에 해당 항목(열) 자체가 부재 | txt(레코드에 키 없음) |

- 실행 화면 사전검증 경고는 배지·게이트와 같은 **‘미입력’**으로 통일한다(과거 ‘값이
  비어 있는 필드’ 2이름 해소).
- txt의 열 부재는 **‘항목 없음’**으로 표기해 실행 화면의 ‘미입력’(출력값 빔)과 구분한다
  (1단어 2개념 해소). txt의 `missing`(코드 심볼)은 유지.

## 3. 수량 단위 분류사 규약 (UD-34)

| 대상 | 분류사 | 어순 |
|---|---|---|
| 레코드·데이터 행·생성 문서 | **건** | `레코드 3건` / `데이터 불러옴 … 3건` |
| 매핑 미리보기의 데이터 위치 | **행** | `이전 행` / `행 1/3` / `다음 행` |
| 등록 작업 실행의 결과 선택 영역 | **문서** | `생성 대상 문서`(내부 데이터 단위 ‘레코드’ 노출 금지) |
| 단일/다중 작업 생성 축 | **작업 수 + 데이터 적용 범위** | `이 작업 실행` / `같은 데이터로 여러 작업 실행` |
| 필드 | **개** | **`필드 N개`**(‘N필드’ 어순 금지) |
| 목록 헤더 카운트(작업·템플릿·데이터셋·매핑 프로파일 목록) | **건** | `N건`(홈·템플릿·풀·매핑 프로파일 통일) |
| 파일·토큰·잔존 토큰 등 이산 요소 | **개** | `파일 N개` / `미변환 N개` |
| 시트 크기 병기(시트 선택 다이얼로그, T2) | **행 / 열** | `약 N행×M열` — openpyxl 저장 시점 메타 **근사치**(아래 §6), ‘레코드 N건’(논리 데이터)과 다른 물리 격자 개념이라 **‘건’ 금지·‘약’ 필수** |

## 4. 폼·섹션 배치 규약 (UD-41)

- **폼 행은 라벨 열 공유 그리드**(`QGridLayout`) — 라벨 시작선·필드 시작선을 통일한다.
  기준 패턴은 작업 편집기 스텝4(`SaveJobPage`)의 그리드. 위저드 스텝1·2도 이를 따른다.
- **동급 섹션은 동일 프레이밍**(카드/그룹박스). 실행·매트릭스 화면의 데이터·생성 대상
  문서(매트릭스는 레코드)·저장 폴더 섹션은 ‘작업 선택’·‘미입력 필드 확인’과 같은 `QGroupBox` 카드로 통일한다
  (섹션명은 박스 제목으로). 레이아웃 변경 시 **위젯 속성명·시그널 계약을 보존**한다
  (스모크 라벨 계약).

## 5. 검수 범위 (RC-26 근본 원인 재발 방지)

정준 용어 치환·단위 규약은 아래 **모든 표면**을 대상으로 한다(정상 경로 라벨만이
아니다). 새 문구를 추가할 때 이 목록을 검수한다:

- 오류/경고 다이얼로그 문구·제목 (`QMessageBox`, `confirm_destructive`)
- 모달·창 제목 (`setWindowTitle`, 다이얼로그 title 인자)
- 지시문·부제 (`setSubTitle`, 안내 라벨)
- 로그·진행 문구 (`_say`, 완료 요약)
- 카드 메타·배지·목록 헤더 카운트
- 파일 IO 버튼·다이얼로그 제목

코드 식별자(내부 심볼)는 검수 대상이 아니다 — **사용자 문구만**.

## 6. 시트 선택 다이얼로그 문구 (T2)

다중 시트 통합문서(xlsx/xlsm)를 겨눌 때 뜨는 확정 다이얼로그
(`gui/view_helpers.py` `ask_sheet_choice` — 5표면 공유: 위저드·실행·매트릭스·txt·풀 등록)의
정준 문구다. 표면별 재작성 금지 — 헬퍼가 단일 출처라 문구 드리프트가 구조적으로 없다.

| 자리 | 정준 문구 |
|---|---|
| 다이얼로그 제목 | **시트 선택** |
| 지시문 | `이 파일에는 시트가 여러 개 있습니다 — 사용할 시트를 확정하세요.` + 줄바꿈 `(행×열은 저장 시점 근사치)` |
| 목록 항목 | `{시트명} — 약 {N}행×{M}열` |
| 확정 후 재진술(파일 라벨, run/matrix/txt 공용 컨트롤러) | `{경로} [시트: {시트명}]` |
| 확정 후 재진술(위저드 스텝2 요약) | `… — 시트: {시트명}` |
| 데이터셋 카드 메타(풀) | `파일: {이름} · 시트 {시트명}` |

- **행×열은 근사치다** — openpyxl read_only 메타(`max_row`/`max_column`)는 파일 **저장 시점의
  dimension 기록** 기반이라 실 데이터 행수와 다를 수 있다(삭제된 행 잔재 등). 그래서 항목
  문구에 **‘약’을 반드시 붙이고**, 지시문이 근사치임을 재진술한다 — 정확한 수치로 오독되면
  "행수가 다르다"는 거짓 경보를 사용자가 스스로 만든다. 로드 후 정확 수치는 ‘레코드 N건’
  (§3, 논리 데이터)이 따로 말한다.
- **‘시트’는 이 개념에만**(§1) — 통합문서 안의 데이터 표 1장. 데이터 풀의 저장 1건은
  ‘데이터셋’, 매핑 대상 열은 ‘데이터 항목’이다(혼용 금지).
- CSV·단일 시트 파일에는 이 다이얼로그가 뜨지 않는다(물을 것이 없음) — 문구도 라벨 병기도
  없다. 결정 근거는 [UI_DESIGN_DECISIONS.md](UI_DESIGN_DECISIONS.md) ADR M.
