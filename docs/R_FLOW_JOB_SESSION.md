## 확정 결정

### 블록 1 — 세션 패널 4존(헤더·데이터·본문·완료)

1. **관문 = 데이터 존.** 세션 진입 시 포커스는 "첫 미답 질문"(진입 경로별 분기 규칙 0 —
   경로가 이미 답한 질문은 건너뛴 곳에 선다).
2. **선택 모델 = 열거형.** 행 클릭 토글 + **Shift 범위**(앵커 행의 상태를 범위에 전파).
   Ctrl 은 오범주화로 어휘에서 제거(토글이 이미 개별 선택). "필터=선택" 발명은 기각.
3. **선택은 필터를 관통한다.** 필터는 보기만 바꾼다 — 필터 밖 선택은 하단 고정 스트립으로
   상시 가시(1부 결정 6 "관통 가족"의 3번째: 필터·경보·선택). 슬롯이 총계를 상시 진술한다.
4. **「전체 선택」+「전체 해제」.** 구 「매치 전체 선택」에서 개명(블록 4 우회) — "매치"의
   담보는 버튼 이름이 아니라 게이트 정의줄 재진술이 진다(정의-유래 선택은 정의줄을 재진술,
   "정의 없음" 상태는 필터 계약으로 소멸).
5. **게이트 = 선택 스냅샷 재진술.** 소량은 전부, 대량은 표본 + "⋯ 외 N건" 펼침. **표본
   추출은 가지별 층화**(라운드 종결 확정 2026-07-19: 광의 OR 정의에서 소수 가지의 매치가
   반드시 표본에 등장 — "표본 뒤에 숨는 오버매치"가 새 경보 없이 구조적으로 소멸. 원 문안
   "대량 표본"의 개정이며, 평문 진술(가)·가지 강조+프루닝 권유(나)는 이로써 불요).
6. **본문 존 = 라이브 거울, 의식은 「생성」 1회.** 중간 승인 버튼 없음 — 이미 보이는 것을
   재검증하지 않는다. 거울의 형상은 블록 6 결정 36(필드 채움 테이블)이 완성.
7. **완료 존 = 세션 스코프**(F29 승계). 역사는 호버 툴팁(1부 일화 문법 재사용).
8. **선택·필터 정의 = 세션 휘발, 작업 무기억.** 행 선택·필터 정의를 작업에 저장하지 않는다
   (명문 결정). 예외 하나: 작업 전환 시점의 **인계용 값 객체** 전달(블록 4 본안 결정 24 —
   저장이 아니라 전달이라 무기억 비침해).

### 블록 4 — 필터 선언 모델(우회) · 세션 가드·건 연속성(본안)

23. **검색↔필터 단일 기제 계약.** 화면 편집 주체 = 필터 상태 하나. 전열 검색 = 재현 OR 그룹
    설치(실매치 열만 가지·가지 프루닝), 검색창 = 그룹 편집기. 열-조건은 열 머리 소유(같은 열
    OR·열 간 AND — 엑셀 동형). **자모(초/중/종성) 분해 부분일치**(산술 분해 + 겹받침 성분
    확장·음절 역매핑 하이라이트). **(빈값)은 일급 값**(없으면 여집합 표현 불가 — "조용한
    빈칸 금지"의 필터 판).
24. **필터 정의 스코프 = 세션 생존.** 같은 행 재방문·레일 이동에도 생존, 작업 전환·앱 종료
    시 소멸. 단 전환 시점에 인계용 값 객체로 전달 가능(결정 8의 예외 — 저장 아닌 전달).
25. **필터 입력기 = B 엑셀식 아이콘 펼침** *(라운드 종결 확정 2026-07-19)*. 열 머리 아이콘
    패널에 열별 부분일치 검색 + 값 체크리스트(같은 열 OR) 동거 — 확정된 필터 모델을 담을 수
    있는 유일 입력기(별도 필터 행 A는 다중 OR·부분일치·(빈값)을 드롭다운 하나에 못 담음).
    엑셀 학습효과 상속. 발견성 약점은 칩 줄 + 게이트 정의줄 재진술이 보완. 경합은 입력기의
    형태였고 상태 모델이 아니다.
26. **Explorer 문법 기각 확정 + 파괴 전이 원장.** 체크박스 시각 문법(결정 13)이 토글 자연
    기대의 답 — 오클릭 전멸 전이의 상류 제거, Ctrl 어휘 부활 없음. 가드 대상 = T1(행 전환)·
    T3(txt 큐 부분 진행) · T2(마법사 인계) = 비파괴화(무장 시 자동 선택 생략 + 조용한 고지
    한 줄) · T4(전체 해제) = 명시 동사라 가드 불요 · T5(앱 종료) = 수용 손실. 소거 판정:
    오클릭 토글 자가복구·필터 변경(선택 관통)·전체 선택 가산적·Shift 대칭 복구.

    생성·완주 집합 ∨ 미소비 수동 필드 입력 ∨ 큐 0<M<N). 완료 이벤트 = 무장 해제(내역은
    완료 존이 담보). modal.js 이진 확인 + 종류별 수치 재진술(기본 포커스 = 머무르기·Escape =
    머무르기). 프루닝은 술어 불포함(경계 = 재진술 가능성 — 소실 창은 결정 28 재적용이 복원).

    뒷문이라 기각 — 교차 재사용은 재타이핑 몇 초, 끼어들기 소실 수용 = 직전성이 연속성의

    게이트 승계). 템플릿 = **세션 사본 + 라이브 편집**: 라이브러리 유래도 사본이라 두 출발
    모두 원문 편집 탭에서 자유 수정(타이핑이 토큰 폼 실시간 재구성), 편집 순간 「(수정됨)」
    정직 강등(칩-라이브 동형), 역반영 자동 없음 = 명시 승격만(동명 덮어쓰기 확인 게이트),
    수정 템플릿 = 새 기안 가드 성분 합류.
35. **지속성 축 비대칭 3호(원리 기록).** 정규 기안 세션 = 안정화된 레시피의 소비 표면(편집·
    소비 분리 — 편집은 에디터 소관) vs 빠른 기안 = 안정화 이전이라 편집·소비 미분화가 본질.
    1호 = 제스처 매핑(결정 30), 2호 = 완화 조항(결정 31), 3호 = 세션 사본 라이브 편집(결정 34).
    *(독법 확정 2026-07-21, R-info 3부)*: 이 결정은 **성질 진술이지 거처 진술이 아니다** —
    두 성질이 한 화면에 살아도 비대칭은 유지되며, 판별자는 화면이 아니라 **세션 유래**다
    (목록 선택 = 안정화된 레시피 소비 / 템플릿 붙여넣기 = 편집·소비 미분화). 합병 설계는 이
    비대칭을 지우지 않고 유래로 옮기는 것이 성립 조건. 부수 정정: "편집은 에디터 소관"의
    에디터는 블록 2 개정(결정 39~41)으로 사망했다 — 지금 그 자리는 **상세 패널 편집 모드**다.
