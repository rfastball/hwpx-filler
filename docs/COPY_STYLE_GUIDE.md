# 화면 문안 스타일 가이드 (v1 확정, 2026-07-22)

> **문서 상태:** 현재 정본
> **권위 범위:** 사용자 대면 문장의 문형·길이·금지 표현
> **후속 정본:** 없음
> **편집 정책:** 계속 갱신

전수조사(UI_COPY_AUDIT.md)의 처방을 규칙화한 것. 이후 모든 화면 문안이 따른다.

## 1. 대원칙

**문안은 두 가지만 말한다: ① 무엇이 됐는가(안 됐는가) ② 다음에 뭘 하는가.**

- 설계 이유("반사적 확인 취소를 막기 위해"), 시스템 원칙("조용히 첫 시트를 쓰지 않습니다"), 안 한 일의 나열("데이터를 바꾸지 않았습니다")은 **쓰지 않는다**. 원칙은 동작(묻기·차단)으로 지키는 것이지 문안으로 낭독하는 게 아니다.
- **예외**: 파괴 확인 모달의 손실 집합 재진술("사라지는 것: 선택 3행 · 필터 2조건")은 계약이다. 유지하되 목록형으로 압축한다.

## 2. 문형별 규칙

| 문형 | 상한 | 골격 |
|---|---|---|
| 버튼·메뉴 | 명사(구)·동사구 | `삭제`, `덮어쓰고 생성` |
| 상시 힌트·화면 부제 | **1문장** | 무엇을 하는 곳인지만 |
| placeholder | 1구 | 입력할 것의 예 또는 이름 |
| 툴팁 | 1문장 | `이름: 설명` (— 금지) |
| 상태·결과 | 1문장 | 결과만. `저장했습니다: 이름` |
| 오류 | **최대 2문장** | ① 문제 ② 조치. 원인 나열·해명 금지 |
| 확인 모달 | 제목=행동 / 본문=결과·손실 목록 / 버튼=동사 | 손실은 `사라지는 것: A · B` 압축형 |

## 3. 표기 규칙

1. **em dash(—) 전면 금지** (문장 안에서). 대체 우선순위:
   ① 뒤 절 **삭제** (절반 이상이 없어도 됨) → ② 마침표로 두 문장 → ③ 쉼표·괄호 → ④ 툴팁·라벨의 `이름 — 설명`은 `이름: 설명`
   - 예외: 테이블 빈 칸 마커 단독 `—` 는 유지 (문장이 아님)
2. **낫표 「」 → 작은따옴표 '…'** 전면 교체. `〈빈 값〉` 같은 홑화살괄호도 `(빈 값)` 계열로.
3. 명령형은 **`~하세요`** 로 통일 (`~해 주세요`, `~해야 합니다` 지양).
4. 개수·건수 병기는 현행 유지 (`선택 3/120 · 표시 45`).

## 4. 용어 정본표

### 4-A. 은어 → 일상어

| 현행(화면 노출분) | 정본 | 비고 |
|---|---|---|
| 결속하다 / 물리다 / 겨누다 | **연결하다** | "다른 데이터를 겨누면" → "데이터를 바꾸면" |
| 완주 | (삭제) | `완주 — 3건 전부 복사` → `3건 모두 복사했습니다.` |
| 본문 · 거울 (존 제목) | **채울 값** | 「이 문서에 들어갈 값」의 이름 |
| 매체(HWPX·TXT) | **형식** | |
| 게이트 | (풀어쓰기) | `게이트를 통과해야` → `아래 항목을 확인해야` |
| 표면화 | (삭제·풀어쓰기) | `빈 값 표면화를 확인하세요` → `빈 값 목록을 확인하세요` |
| 드리프트 | **템플릿 변경** | 결과 줄 라벨 |
| 프루닝·잔존·되읽기·결합자·피연산자 | 풀어쓰기 | 개별 문안에서 처리 |
| 루트 밖 세션 템플릿 | (풀어쓰기) | `이 세션에서만 쓰는 템플릿입니다` |

### 4-B. 동의어 → 정본 (같은 개념은 전 화면 같은 말)

| 개념 | 정본 | 폐기 |
|---|---|---|
| 데이터의 세로줄 | **데이터 열** | 헤더·컬럼·데이터 항목·소스(표 머리에서) |
| 값이 비어 있음(데이터 쪽) | **빈 값** | 미입력·빈칸 (혼용분) |
| 의도적으로 비움(사용자 확정) | **비움 확정** | 비워둠(선언)·확정-비움·빈칸 선언·(의도적 빈칸) |
| 토큰에 맞는 열이 없음 | **항목 없음** | 데이터에 없음·미매칭 (설명문 안 혼용분) |
| 자동 제안 없음 | 후보 없음 | (유지 — 별개 개념) |

### 4-C. 화면·기능 이름 (변경 없음, 표기만)

`'작업'`, `'기안'`, `'맞추기'`, `'그룹 없음'`, `'가져오기…'` — 낫표만 따옴표로.

## 5. Before → After 표본

| 위치 | Before | After |
|---|---|---|
| 시트 모달 | ~에 시트가 여러 개입니다 — 어느 시트로 채울지 확정하세요. 조용히 첫 시트를 쓰지 않습니다. | ~에 시트가 여러 개입니다. 채울 시트를 선택하세요. |
| 사전검증 | 검증 완료 — 문서를 생성할 준비가 됐습니다. | 검증 완료. 생성할 수 있습니다. |
| 기안 완료 | 완주 — 3건 전부 복사했습니다. | 3건 모두 복사했습니다. |
| 기안 빈 상태 | 데이터를 물리면 여러 건을 한 번에 채웁니다 — 안 물려도 아래 「맞추기」에서 직접 채워 복사할 수 있습니다. | 데이터를 연결하면 여러 건을 한 번에 채웁니다. 없이도 '맞추기'에서 직접 채울 수 있습니다. |
| 매핑 부제(4문장) | 데이터를 고르면 컬럼·표본이 아래 표에 그대로 차오릅니다. 자동 제안은 초안이니 모든 행을 검토·확정해야 저장으로 진행합니다. 채우지 않을 필드는 소스를 (비움)으로 두고 확정하세요. 작업엔 데이터가 저장되지 않습니다 — 매핑 검토용 샘플입니다. | 필드마다 데이터 열을 지정하고 전 행을 확정하세요. |
| 거울 안내 | 이 문서에 들어갈 값입니다. 미입력 필드 행은 반사적 확인 취소를 막기 위해 직접 눌러 확인해야 생성이 열립니다. | 이 문서에 들어갈 값입니다. 빈 값 행은 눌러서 확인해야 생성할 수 있습니다. |
| 데이터 등록 부제 | 경로 참조만 저장합니다 — 행·내용은 저장하지 않고 실행할 때 다시 읽습니다. | 파일 위치만 기억하고, 실행할 때 원본을 읽습니다. |
| 저장 실패(부분) | 다만 실행 기록을 남기지 못했습니다({e}) — 문서는 모두 만들어졌지만 홈의 최근 실행 이력에는 이번 실행이 빠집니다. | 문서는 모두 만들어졌지만 실행 기록 저장에 실패했습니다({e}). |
| 파괴 confirm | 직전에 붙여넣던 세션에서 이미 N건을 복사했습니다 — 되돌릴 수 없습니다. 이 사본이 「이번 세션」 자리를 대신합니다. 붙여넣던 세션의 원문 편집·데이터·선택·복사 진행은 저장된 기안에 보관되지 않아, 사본으로 가르면 함께 사라집니다. | 사본으로 편집하면 붙여넣던 세션이 사라집니다.\n사라지는 것: 원문 편집 · 데이터 연결 · 선택 · 복사 진행 N건(복사한 결과는 남음) |

## 6. 확정 결정 (2026-07-22)

1. **파괴 확인 버튼**: `머무르기` → **`취소`로 통일**. 파괴/일반 구분은 확인 버튼의 동사(`삭제`, `덮어쓰고 생성`)와 기본 포커스=취소가 담당한다.
2. **"미입력" 어휘**: **`빈 값`으로 전면 통일** (게이트 문맥 포함). 빈 상태 어휘는 `빈 값`(데이터 쪽) / `비움 확정`(사용자 의도) / `항목 없음`(맞는 열 없음) 셋뿐이다.
3. (개정) `〈빈 값〉` 미리보기 마커는 **유지** — 채운 본문 안에서 실제 내용과 구분되는 표지가 필요하므로 3-2의 예외.

## 7. 적용 범위·불변

- 코드 식별자·JSON 키·테스트 로직 불변, **문안 문자열만**.
- 코드 주석·docstring의 em dash는 범위 밖(사용자 안 보임).
- 문안 어서션 테스트는 문안과 같은 PR에서 동수정.
