## 프레이밍

발단은 사용자 발제 **"작업의 집이 없다"** — 템플릿·데이터는 전용 관리 화면이 있는데 중심
개체인 작업만 없어, 관리 동사가 대시보드 카드에 무단 거주(개체-거처 비대칭). 토의는 추상화
질문("그 일은 어떻게 정의되고, 어떻게 머릿속에 살며, 어떻게 탐색되는가")과 작업 시나리오
추적(단골 건 하나·두 출력의 하루)으로 진행했고, 모든 결정은 그 추적에서 검증됐다.

핵심 실증(사용자 확인): 트리거는 **데이터-우선**(건이 먼저 오고 문서가 따른다) · 단골은
**금액구간 변형 군집**으로 갈린다(변형 선택은 기억이 아니라 데이터 대조) · 같은 건에서
**hwpx·txt 동시수요 확실** · 규모는 통상 10~20, 최대 30~40(+기안 ≈ N=50 상한) ·
데이터 갱신은 외부(대장 파일)에서 일어나 전달된다.

## 확정 결정

1. **정체성·이름**: 대시보드 → **「작업」**(C안), 부팅 랜딩. 멘탈 모델 = 작업이 중심,
   템플릿·데이터 관리는 **작업을 위한 자원**(3관리 대칭 명명은 기각 — 작업은 관리 대상이
   아니라 중심 개체).
2. **개체 통일**: 작업 = 레시피(템플릿+매핑), **출력(hwpx/txt)은 속성**. txt 기안 작업은
   **명시적 개체**(템플릿 재간판 아님 — F41 관리층 단일 원칙). 표시형은 매핑의 소유라
   기안도 작업 경유 시 서식을 얻는다(`cli.py:169`가 증명한 코어 경로의 GUI 착지, F36 서식갭
   소멸 지점). 일회성 기안(저장 없는 지름길)은 레일 「기안문 채우기」에 존치 — 지속성 축 직교.
3. **형상 = 변형 B(상시 master-detail)**: 좌 목록(두 구획: 문서 작업·HWPX / 기안 작업·TXT,
   구획 헤더에 열 틴트 파랑/초록 승계) + 우 **4존 세션 패널**(헤더·데이터·본문·완료).
   행 클릭 = 패널에서 in-pane 세션, **화면 전환 0·레이아웃 불변**(전환 모델 기각 → 애니메이션
   문제 원천 소멸). 선택 전 패널은 조용한 빈 상태. 파급: 레일 「실행」 잉여화(제거 시점은
   R-flow), 실행 화면 작업 드롭다운 소멸 예정.
4. **회수 문법**: 과업은 known-item 회수(자기 레퍼토리) — 패싯은 탐사 도구라 부적합(F8 재확증).
   정렬 3종(**기본 이름순** — 군집 인접이 데이터-조건부 변형 선택을 가능케 함 · 선택 영속 ·
   목록 전체 공통 단일 컨트롤) + 타이핑 필터. **그룹 배열은 안정**(이름순 고정, 「그룹 없음」
   마지막; 정렬 키는 그룹 내부에만 — 대표값 그룹 재배열은 CC 이슈 #70104의 "그룹이 뛰어다닌다"
   실전 반증으로 폐기).
5. **간단 그룹화**: 작업당 **선택적 단일 그룹**(축·DSL·렌즈 없음 — ⑥ 다축 태그 제거와 구분).
   「그룹 없음」 명명 접이식 구획 · 들여쓰기 26px + 그룹 간 여백 > 행 간 여백(위계 2축) ·
   접힘 화살표 이름 오른쪽·호버 노출(단 **접힌 그룹은 상시 노출**) · 지정은 ⋮ 「그룹으로 이동 ▸」
   서브메뉴(기존 그룹 + 새 그룹 + 해제). **퇴화 불변식: 그룹 0개면 헤더·들여쓰기 없는 평면.**
6. **접힘-펼침 정책**: ①마지막 상태 영속 — **Python 설정**(webview 저장소 금지, #74 전례)
   ②무상태 기본 = 전부 펼침, 새 그룹 = 펼침 ③**필터는 접힘 관통**(매치 그룹 임시 펼침, 해제 시
   영속 상태 복원) ④**경보는 접힘 관통** — 자동 펼침 대신 접힌 헤더에 ⚠ 표지 ⑤선택 행의 그룹을
   접어도 선택 유지. N=50 감당의 주역은 그룹화가 아니라 **접힘의 영속**(접힘 정상상태 ≈ 한 파).
7. **행 문법**: 행 면 = **이름 단독**. 행 클릭 = 주동사(출력별 세션 진입). ⋮ 호버·포커스 노출:
   편집 / 복제 / ─ / 그룹으로 이동 ▸ / ─ / 삭제(빨강, 재진술 확인 모달; modal.js — 네이티브 금지
   F7 교훈). **경보 행 예외**: 템플릿 끊긴 행은 경보 틴트 + 「템플릿 다시 연결…」 상시 노출,
   행 클릭도 재연결로(경보 어포던스는 숨지 않는다).
8. **정보 위생**: KPI 소멸(실재≠유용, F13) — 경보만 승계(손상 화면급 + 템플릿-없음 조건부).
   「이어서 실행」 패널 소멸(최근순 강요의 축소판 — 어제 변형이 오늘 구간을 선점 금지).
   일화(마지막 실행 시각+건 요약, F33 재사용) = **호버 툴팁 강등**. 개수 표기 = **접힘 "(N)"만**
   (펼침 개수는 KPI와 같은 과시 함정 — 과시/표지의 경계 = 접힘 여부). 관리 상세(템플릿 경로·
   데이터 참조 등)는 별도 뷰 없이 **편집이 겸용**.
9. **태그·패싯·group-by 완전 도려냄**: 데이터 모델까지(matrix 전례), 하위호환 무배려(잔존
   tags 키 운명 무배려). F7 네이티브 프롬프트 통째 소멸.
10. **창작**: 구획마다 ＋(「＋ 새 작업」/「＋ 새 기안 작업」 — 매체 맥락 상속, 기안 ＋는
    라이브러리 txt 선필터). 에디터는 출력-불가지: 1단계 라이브러리에서 고른 **템플릿의 매체가
    곧 출력 속성**(별도 출력 선택 단계 없음).
11. **txt 세션 의미론**: 데이터 **1행**(기안은 단건) · 매핑 선채움(표시형 서식) + 라이브 잔여
    필드 + 실시간 미리보기(미채움은 `{{필드}}`로 정직 노출) · **복사 = 완료 이벤트**(last_run_at
    스탬프 — 최근 실행순·일화의 txt 의미론) · 게이트 = **빈칸 경보**(산출물 무게에 비례:
    hwpx 강제확인 게이트의 기안판, 조용한 빈칸 금지).

## 기각 대안 아카이브(근거 요약)

- **A안**(대시보드 이름 유지)·**B안**(관리 화면 신설, 레일 7) — C안이 흡수(위장 크롬 제거 +
  레일 불증가). **「작업 관리」 명명** — 3관리 대칭보다 중심 개체 격상이 모델에 정합.
- **인터리브 단일 목록**(출력 배지 구분) — 제출처 맥락 분리 훼손, 사용자 기각 → 두 구획.
- **v6 두 열 + 화면 이동** — 변형 B에 패배(짝-날 전환 0·폭 문제의 구조적 해). 가시 행수
  절반(32→16)은 정직한 트레이드로 수용.
- **전환 모델**(두 열 ↔ master-detail 모프) — 이중 기하로 위치 근육기억 파괴 + FLIP 애니메이션
  실비용. 상시 모델로 원천 회피.
- **대표값 그룹 재배열**(정렬 양층 적용) — CC #70104 실전 반증("그룹이 뛰어다닌다")으로 폐기.
- **반대편 구획 자동 접음**(사용자 제안 후 철회) — 짝-날 교차 흐름과 충돌; 영속 수동 접힘이
  같은 효과를 사용자 손으로.
- **호버 미리보기 패널** — 보여줄 내용이 이미 다른 거처(이름·툴팁·편집)에 배정, 깜빡임·접근성
  비용만 잔존. 실행 세션은 입력을 가진 작업공간이라 호버와 원리적 양립 불가.
- **이름 접두 자동 그룹 파생** — 구분자 추측 = 조용한 추측. **고정(Pin)·보관(Archive)·수동
  그룹 배열** — 수요 실증까지 유보(보관은 데이터셋 archived 전례 있음, 코퍼스 노화 시 재론).
- **이어서 실행 유지·열 머리 개수·상시 동사 버튼·호버-노출 일급 동사·선택+툴바** — 본문 결정
  8·7의 근거로 각각 기각.
- **작업 쌍 객체** — 임의적·과잉. 짝짓기는 두 구획+사람 눈. 대체 아이디어 **「동일 데이터
  소스 작업 묶음 일괄 생성 즐겨찾기」** = 향후 후보(opt-in 단축이라 matrix 제거와 비모순,
  착수 조건 = 짝-날 행 재선택 마찰의 실사용 실증; 채택 시 행 1회 선택으로 쌍 불일치도 원천 봉쇄).

## 외부 검증(CC 이식)

Claude Code 데스크톱 사이드바 = 같은 과업 형상(자기 레퍼토리 known-item 회수)의 검증된 해 —
행 클릭 주동사·호버 ⋮·커스텀 그룹·「그룹 없음」 구획·들여쓰기 위계를 이식. 단 **시간성 문법은
불이식**(세션=비행 중 임시 작업 vs 작업=안정 레시피: 최근순 기본·읽지 않음·자동 보관은
우리 개체에 부적합). 근거: 공식 리디자인 블로그(claude.com/blog/claude-code-desktop-redesign,
"stays focused on what's live") · 공식 문서(code.claude.com/docs/en/desktop) · 이슈 #70104·#72126.

## R-flow 인계 목록

1. 세션 패널 4존 내부 상세 설계(hwpx 행 선택 표·파일명 미리보기·게이트 배치, F20 데이터-우선
   흐름과 접합).
2. **세션 가드**: 입력 진행 중 다른 행 클릭 시 재진술 확인(editJob 미저장 가드와 대칭).
3. **건 연속성**: 공고서 실행 → 기안 세션이 "방금 그 행"을 이어받는 문제(행 재선택 마찰·오행
   위험 — 기록만, 기제는 R-flow에서; 묶음 즐겨찾기가 흡수 후보).
4. 「기안문 채우기」 레일 항목의 패널 통합 여부(일회성 기안도 패널에서? — 존치가 기본값).
5. 레일 「실행」 제거 실행 시점.
6. txt 완료 이벤트(복사 시 last_run_at 스탬프) 구현 의미론.

## 2부 잔여(R-info 종결 조건)

템플릿 관리 실화면 순회 → F12 세부: 에디터 1단계 라이브러리-우선·자유 파일선택 import 강등·
F16 버튼 재설계·F41 인앱 txt 저작의 폴더 착지·템플릿 관리 두 카탈로그의 평행 문법 정렬
(폴더트리 그룹은 템플릿에만 — 기질이 사용자 폴더라서; 작업 그룹과 혼동 금지).
