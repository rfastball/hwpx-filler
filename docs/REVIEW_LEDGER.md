## 부록 C — ST-33 재분류 (조치 중 발견)

**ST-33**(diff 구판=신판 동일 파일 가드)은 조치 착수 시 **RC-32와 충돌**함이 드러나 재분류했다(반영 커밋 `6f209fc`). `hwpxdiff`는 이미 RC-32로 변경 0건을 `NO_CHANGES_MESSAGE`+KPI(0/0/0/0)로 정직하게 확정한다(회귀 테스트 `test_diff_zero_changes_shows_shared_no_changes_copy`가 고정). 즉 발견이 우려한 '무의미 비교 조용한 확정'은 RC-32가 이미 완화했고, 별도 차단 가드는 그 의도된 zero-changes 흐름과 정면 충돌 → 가드 철회, **기각 계열(기존 설계로 완화)**로 재분류. ST-34(최근 비교 경로 툴팁)는 충돌 없어 착지. **교훈**: 소스 기반 리뷰가 놓친 인접 결정(RC-32)은 조치 착수(코드 접촉) 시 드러난다 — 검증 단계에 회귀 코퍼스 대조를 더 실으면 조기 포착.
