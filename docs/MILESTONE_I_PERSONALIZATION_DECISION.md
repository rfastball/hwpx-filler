# 마일스톤 I 설정 존중·개인화 결정 (#221)

## 브라우저 확대

`pywebview.create_window(..., zoomable=False)`를 명시적으로 유지한다. 브라우저 확대는 앱의
고정 최소 창 크기, 모달 좌표, 내부 스크롤포트까지 임의 배율로 갈라 재현성을 낮춘다. 대신
모든 `--fs-*` 토큰을 `rem`으로 생성하고, 저장형 앱 배율 `기본(100%) / 크게(125%) /
더 크게(150%)`를 제공한다. 이 배율은 `settings.json`에서 부팅 전에 되읽어 창을 표시하기
전에 적용한다.

## 선택과 직접 조작

창은 `text_select=True`로 만들어 본문 텍스트를 선택·복사할 수 있다. 레일, 버튼·선택 상자,
작업 데이터 행만 `user-select:none`으로 제한한다. 데이터 행의 Shift+클릭 범위 선택은 클릭
이벤트 계약이므로 그대로 유지된다.

작업·기안 좌 목록 폭은 키보드 조작 가능한 세로 스플리터를 채택한다. 두 화면은 같은
`master_width` 값을 공유하고 180~420px로 제한한다. 820px 이하 적층 레이아웃에서는 스플리터를
숨기고 목록 폭을 자동으로 되돌린다.

## 영속과 복구

글자 배율, 레일 접힘, 목록 폭, 창의 정상 크기·위치·최대화 상태는 오리진 비의존
`settings.json`에 저장한다. 저장 창의 제목줄이 현재 가상 데스크톱과 만나지 않으면 위치·크기를
기본값으로 폴백하여 모니터 분리 뒤 화면 밖에 갇히지 않게 한다.
