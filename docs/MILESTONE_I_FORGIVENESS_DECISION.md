# Milestone I — 관용 장치 결정 기록 (#220)

## 삭제 정책

작업과 템플릿 삭제는 **루트 하위 `.trash`에 30일 보존 + 화면별 최근 1건 즉시 복원**으로 정했다.

- 일반 목록은 `.trash`를 스캔하지 않으므로 삭제 직후 항목은 사라진다.
- 새 삭제는 UI의 1슬롯 복원 대상을 덮지만, 이전 파일도 30일 동안 휴지통에 남는다.
- 원래 경로에 같은 이름이 생기면 덮어쓰지 않고 복원을 거부한다.
- 열린 세션의 복구 불가능한 선택·진행이 사라질 때만 사전 확인을 유지한다. 클린/무관 세션과
  홈·템플릿 관리 삭제는 확인 대신 사후 `되돌리기`를 제공한다.

휴지통 방식을 택한 이유는 앱이 비정상 종료돼 메모리의 1슬롯을 잃어도 파일 자체는 남고,
확인 클릭스루를 줄이면서 영구 삭제와 이름 충돌을 명시적으로 분리할 수 있기 때문이다.

## 다른 관용 장치

- `모두 해제`: 해제 직전 확정 행 인덱스를 1슬롯에 저장하고 한 동작으로 그대로 복원한다.
- 문서 생성 중단: 현재 문서는 완결하고 다음 레코드 경계에서 멈춘다. 완료·실패·미착수를 분리해
  결과에 쓰며 완료 파일은 유지한다.
- TXT 편집 모달: 최초 내용과 달라진 경우에만 취소/Escape에서 폐기 확인을 한다.
- 입력 검증: 태그·개명·기안 저장명 오류는 입력을 보존한 채 같은 표면에 인라인 표시한다.
- 확인 감축: 클린 편집 종료는 즉시 통과하고, `사본으로 편집`의 두 확인은 한 번으로 합쳤다.

정적 `Modal.confirm` 호출은 감사 원장 38곳에서 37곳으로 줄었고, 조건부 확인의 런타임 발화도
클린 상태에서 추가로 줄었다. 덮어쓰기·제자리 변환처럼 복구 불가능한 정확성 게이트는 유지한다.
