## 본질 / 스코프 — 값 치환의 확장이지 문서 조립이 아님

도구의 본질은 **값 치환의 확장**이다. 다음은 **명시적 non-goal**:
- 조건부 조립(if-then 섹션)
- 문서 레벨 서식 조립
- 단일 문서 병합(N레코드 → 1문서 N섹션)

값 변환(날짜·금액 서식)·표 행복제는 경계선상 — 아래 명시성 원칙으로 판정(명시 선언 + 미리보기면 허용).

**관통 원칙 — 명시성(explicitness).** 파서 철학("충실도 완전·누락은 시끄럽게")을 저작으로
확장: **변환은 명시적 선언 · 암묵 자동 금지 · 결과는 미리보기.** 도구가 타입을 추론해 값을
몰래 재서식하거나 "반복 표처럼 보이니" 알아서 행 복제하는 것은 불쾌선을 넘는다. 스키마의
`inferred_type`은 작성자에게 주는 *제안*이지 자동 실행이 아니다. 컴파일러도 dry-run/미리보기
기본, 명시 확인 시에만 워크북 변형.

## 현 능력 vs 표면 (전 간극 해소)

| 능력 | 코어 | GUI 표면 | CLI |
|---|---|---|---|
| 문서 생성(메일머지) | `batch`/`engine` | 앱 B: 작업 홈·에디터·실행 | 기본 |
| 나라장터 공고 취득 | `data/nara.py` | 소스 선택·키 등록·취득 화면(`gui/nara_view.py`) | `--source nara` |
| 규격서 개정 diff | `hwpxdiff/diff` | 앱 A: 단일 화면 리뷰어(`hwpxdiff/app.py`) | `hwpxdiff` |
| 템플릿 관리(schema/fieldize/lint/drift) | `core/*` | 관리 패널(`gui/template_manager.py`) | 4개 서브커맨드 |

**완료 트랙(상세 = git):** 생성기(B-1~4)·표시형/서식 아크(D-1~7)·매핑+나라장터 취득(B-3·B-3b)·
제품 셸/작업 앵커 데이터모델·exe 패키징(C-1~3)·작업 앵커 다대다 오케스트레이션(ADR J)·
조립 파이프라인(ADR K, petl)·원장/export(ADR L)·컴파일 상태 4-상태 가독성(C)·능동 빈칸 게이트(B)·
누적 치환. 두 능력(문서 생성기 · 규격서 개정 diff)이 공용 토대(`hwpxcore/text_extract`) 위에 얹혀
있고, 직교하는 제품 트랙 C(앱 셸 + 배포)가 감싼다.

## 동결 / 파킹 — 재개 신호 명시

착수 조건(승급 신호)을 못 박아 둔 항목들. 신호 없이 착수 금지.

- **B-5 / 3-① 표·반복 영역 (동결).** 행 수 가변 표(품목·수량·단가) 행 복제
  (`hp:tbl`/`hp:tr`/`hp:tc`). 동결 사유: 공고서 작성 실익 적음. 행복제는 구조 변형
  (rowCnt/cellAddr 재번호·병합 셀·랜덤 ID 중복·행별 필드 스코핑)이라 비용 큼. **재개 선결:
  격리 스파이크로 구현 메커니즘 먼저 확정(한글이 여는가 = 유일 오라클), 라인아이템 데이터
  형태(단일 시트? 메인+하위레코드?) 결정.**

- **D-2 텍스트 반복 행 (동결).** 가변 반복행(품목·수량·단가 라인아이템)의 텍스트 렌더.
  반복 자체는 값싸나 대상 문서군 수요 낮음. 재개 선결 = 라인아이템 데이터 형태 결정(B-5와 공통).

- **D-8 총동원 타입 추천기 (파킹).** 이름 추론만인 현 추천을 값 샘플링·소스 메타까지 확장.
  **책상 위 추정으로는 한계** — 실데이터의 오추천 사례(예: 전부-숫자 계약번호를 amount 로
  오추천) 없이 만들면 "그럴듯하지만 틀린 추천"을 낳는다. **실사용 데이터 확보 후 재개.**

- **법령 개정 오라클 코퍼스 (동결).** 골든(변경탐지)·합성(자작정답)을 넘어 **실데이터 정답
  대비 재현율/정밀도** 검증(테스트 피라미드의 빠진 조각). 필요 파일 3종(law.go.kr, 개정폭 작은
  법령): ① 개정 전 HWPX ② 개정 후 HWPX ③ **신구조문대비표**(정답 오라클 — old|new 2열).
  절차: 추출기 ledger 확인 → 쌍 코퍼스 편입 → 대비표를 정답 변경목록으로 인코딩 → 재현율/정밀도
  테스트 격리 구축. granularity: 개정문(조문 단위)보다 신구조문대비표를 정답 소스로.

- **diff 정밀도 (부분 파킹).** 표 제목 매칭(현 순서 기반)·셀 재번호 인지·코퍼스 폭 확장
  (다른 부류 개정 쌍). 위 법령 오라클 코퍼스가 선행 검증 수단.
