R-flow 시안 — 에디터 매핑 단계 접기

블록 2 토의 시안 v3. 2단계 「데이터 선택 (선택)」을 삭제하고 매핑 단계의 관문으로 인라인 — 행위는 격상, 단계 크롬은 소멸(F18). 칩-라이브 계약 합류: 미확정·미접촉 행은 시스템 소유(라이브 재제안), 수동·확정 행은 사람 소유. F19a=커밋 버튼 소멸, F19b=시간축 접힘, F21=라이브 반영+미리보기 열 압축으로 해소. 동작 실증은 별도 데모 아티팩트 (chip-live-demo). 보라 번호 칩은 하단 결정 범례와 연동.

마법사 4단계 → 3단계

삭제되는 것은 벽이지 내용이 아니다 — 파일 선택·시트 확정·헤더 선택 전부 매핑 단계 안으로 이사한다.

상태 1 — 파일 겨눔 직후 (첫 대면)

선택과 결과가 같은 지면: 파일이 꽂히는 순간 제안이 즉시 돌고, 칩 벽·표·미리보기가 그 자리에서 차오른다. 칩의 일 = 자동 제안·드롭다운 후보·미리보기 열을 세션에서 좁히기(#49). 토글은 즉시 반영되고 커밋 버튼은 없다(F19a 해소). 미사용 칩은 벽에서 빠져 접힌 구역으로, 미리보기에서는 열이 빠진다. 다열 스케일은 「전체 미사용」 후 골라 켜기 + 검색 드롭다운이 감당.

결정 대응 범례

  1. ①F18 — 단계 크롬 소멸: 데이터는 매핑 단계의 관문(머리 자리+초기 포커스). 행위-피드백 동일 지면.
  2. ②F20 — 무데이터는 조용한 opt-out 링크(기본이되 강제 아님 — 등급으로 표현).
  3. ③F19b — 칩 벽 첫 대면 펼침 / 재방문은 개수 한 줄 접힘. 재방문 활성 상태는 저장 매핑에서 파생(#49: 사용=매핑).
  4. ④칩-라이브 계약 — 겨눔 즉시 제안. 미확정·미접촉 행=시스템 소유(활성 헤더 따라 라이브 재제안), 수동·확정=사람 소유. 매핑된 헤더를 끄면 시끄러운 강등(재확정 필요). 커밋 버튼 없음=F19a 해소. 확정↔해제.
  5. ⑤전체 사용·전체 미사용 — 대칭 벌크 쌍. 전체 미사용은 확정 존재 시 차단(#62 정밀화), 실행 시 미사용 구역 자동 펼침(고른다→매핑한다).
  6. ⑥미사용 분리 구역 — 벽에서 완전 이탈, 평시 접힘 "(펼쳐서 다시 사용)". 흐린 칩 혼재 벽은 기각.
  7. ⑦F21 열 압축 — 데이터 미리보기 열=활성 헤더만(미사용 열은 뷰에서 제외, 재활성 시 복귀). 매핑 행은 반대로 숨김 불가 — 의무 잔존(조용한 빈칸 금지).
  8. ⑧검색형 드롭다운 — 다열 스케일 보조(타이핑 필터, 활성 헤더 안에서).
  9. ⑨파일 교체 시 안 맞는 행 in-place 미확정 강등 — 단계 왕복의 유령 상태 질문 소멸.
  10. ⑩F26 — 라이브 예시가 기본 안내. 1행 표본 한 줄 고정 — 전수 나열은 세션 본문 존 소관(소량 전부·대량 표본+펼침).
  11. ⑪RC-20 사전 노출 — 미해소 토큰은 저작 단계부터 경고(게이트 도달 전 정직).
  12. ⑫F27 — 토큰 참조는 전문성축 접힘 뒤(기본 화면 = 입력창+라이브 예시만).
  13. ⑬#33 — 다중 시트 확정도 관문 안에서(시트 칩) — 결과가 같은 지면에서 갱신.
