R-FLOW 블록 3 · 토의 시안 v4 · 합의문 개정 승인 대기

「단건」의 주인은 세션인가, 동사인가

문제인식 2단과 형상 토의를 거쳐 v4에 확정 기울기를 반영했습니다. 선택 = 전-선언 (복사용 렌더링 큐를 만들라는 선언), 고정 작업점(커서가 목록을 걷는 대신 큐가 한 장 카드를 지나감), 처리분 후미 이동(멱등 재복사·완주 감지), 상태 색인(위치·처리·빈칸 지도), 복사 동사는 카드 위(전역 버튼 없음). 시나리오 B를 누르고 카드 우상단 「복사」와 ↓, 색인 점 클릭을 걸어보세요.

1. 긴장: 두 기록이 정반대를 가리킨다

1부 합의문 · 결정 11 (R_INFO_JOB_HOME.md)
txt 세션 의미론: 데이터 1행(기안은 단건) · 매핑 선채움(표시형 서식) + 라이브 잔여 필드 + 실시간 미리보기 · 복사 = 완료 이벤트(last_run_at) · 게이트 = 빈칸 경보.
세션이 단건이라면, 일괄 저장이 설 자리가 없습니다.
순회 원장 · F37 (UX_FINDINGS_101_WALKTHROUGH.md)
일괄 저장 부재(복사/저장 건별만) = 기능 갭. F37 일괄 저장은 트랙 A 배치의 텍스트 판. 하위결정: 분할 vs 라인구분(기존=라인구분) · 일괄 복사는 제외 확정.
배치가 갭이라면, 세션은 단건일 수 없습니다.
문언 충돌: 결정 11의 「1행」과 F37의 「일괄」은 같은 화면을 서로 다르게 정의합니다.
화해안 1단: 「단건」의 주인은 세션이 아니라 「복사」 동사입니다. 기안문의 최종 목적지는 파일이 아니라 내부 시스템의 입력창이고, 클립보드는 본성상 한 번에 하나만 담습니다.
정정 2단: 일괄 저장 자체가 파생 수요입니다. 기안은 어차피 붙여넣어져 내부 시스템에 저장되므로 텍스트 산출물을 소유할 이유가 없습니다. 필요한 것은 일괄 저장이 아니라 건별 복사-붙여넣기 루프의 마찰 제거입니다. 이에 따라 라인구분 vs 분할·일괄 빈칸 경보 미결은 소멸했습니다.
형상 3단(v4 확정 기울기): 전-선언 큐가 고정 작업점을 지나갑니다. 레코드 선택은 복사 선언이 아니라 전-선언 — 복사용 렌더링 큐를 만들라는 선언입니다. 선언이 아니므로 카드 위 「복사」와 우선권을 다툴 수 없고, hwpx와의 문법 대칭이 복원됩니다(선택=전-선언 → 재진술/실물 확인 → 선언; 차이는 선언의 입도뿐 — 집합 1회 「생성」 vs 건별 반복 「복사」). 프레젠테이션은 반전 합성안: 커서가 목록을 걷지 않고, 큐가 한 장 카드(작업점)를 지나갑니다. 처리분은 색인 후미로 이동해 멱등 재복사와 완주 감지를 얻고, 상태 색인이 위치·처리·빈칸의 지도가 됩니다. 위치는 앱이 소유합니다: 다음 = 항상 첫 미처리(블록 1 「포커스=첫 미답 질문」의 큐 판).

2. 걸어보기: 고정 작업점 루프

데이터 존에서 행을 고르면(전-선언) 큐가 만들어지고, 작업점 카드에 첫 미처리 기안이 실물로 섭니다. 복사는 카드 우상단에서. 복사된 기안은 색인 후미로 넘어가고 다음 미처리가 자동으로 작업점에 올라옵니다.

시나리오
데이터 존 (전-선언)
공고번호공고명추정가격납품기한처리
전-선언행 클릭은 복사 선언이 아니라 큐 편성입니다 (블록 1 열거형 선택 승계 · 필터는 보기만 · 선택은 필터를 관통). 복사 의도는 오직 작업점 카드 위에서만 선언됩니다 — 우선권 질문은 어휘로 소거되었습니다.
슬롯
작업점 (고정 · 큐가 지나감)
붙여넣는 곳의 표준 글꼴 · 전역 설정으로 저장됩니다
행을 선택하거나 시나리오 버튼으로 시작해 보세요.
형상원문 렌더는 코드블록 문법입니다. 클립보드는 글꼴을 운반하지 않으므로(글꼴=목적지 소유) 약속은 글자 그대로이고, 배치는 대상 글꼴 선언이 맡습니다: 굴림체·돋움체(고정폭)·맑은고딕(비례폭)이면 공문 타이포가 사실상 전부 커버되며, 넘기려는 글꼴로 미리 보므로 그대로 넘어가는 감각이 성립합니다. 린트는 선언-조건부: 비례폭 선언에서만 연속 공백 정렬을 경보하고 전각 공백 치환을 제안합니다(한글·전각은 전 글꼴 균일폭이라 견고). 표시형 서식(결정 11)은 값의 문자열 형식이라 원문에 포함됩니다. 키보드: ↓/↑ 큐 이동 · Enter 복사. 빈칸 게이트는 블록에 결속, 색인의 노란 점=빈칸·초록 점=처리 후미. 「복사됨」 라벨은 앱이 아는 사실만 주장합니다.
승인 동사 (전역)
정정 2단에 따라 강등. 눌러서 근거를 확인하세요.
입도전역 「복사」 버튼은 없습니다. 선언의 입도가 동사의 위치를 정합니다: 집합 1회 의식(hwpx 「생성」)은 동사 존에, 건별 반복 동사(txt 「복사」)는 대상 위에. 일괄 복사 제외(F37 확정)는 그대로 섭니다.
완료 존 (세션 스코프)
  • 아직 완료가 없습니다.
인계 6산출이 실제로 나간 순간만 완료입니다: 복사가 last_run_at을 찍고(작업 단위), 큐 완주는 조용한 한 줄로만 알립니다 (정상은 조용히).

3. 확정 대기 목록과 잔여 미결

✓확정 기울기 (다음 세션 합의문 개정 승인 대상)

  • 전-선언 어휘: 선택 = 복사용 렌더링 큐 편성 선언. 선택=큐 확정, 우선권 질문 소거.
  • 결정 11 정밀화 문안: 「데이터 1행」 → 「세션은 큐(N행)를 전-선언으로 담고, 동사 「복사」가 작업점에서 1건씩 소비한다」. 블록 2 기록 「txt N=1」 문구 동반 수정.
  • 공유 스테이지 대칭 문안: 두 트랙 모두 선택=전-선언 → 재진술/실물 확인 → 선언. 차이는 선언의 입도(집합 1회 vs 건별 반복)와 재진술 매체(스냅샷 vs 실물)뿐.
  • 합성 프레젠테이션: 고정 작업점(한 장 카드) + 처리분 후미 이동(멱등 재복사· 완주=미처리 소진) + 상태 색인(위치·처리·빈칸). 파생 지정 = 항상 첫 미처리.
  • 복사 동사 = 카드 우상단(전역 버튼 제거) + 키보드 Enter 경로 + 빈칸 게이트 카드 결속.
  • 복사=전진 옵션 기본 꺼짐 · 「복사됨」 정직 라벨(붙여넣음을 주장하지 않음).
  • 대상 글꼴 선언: 클립보드 평문은 글꼴을 운반하지 않으므로 약속은 「글자 그대로」로 정정하고, 배치는 전역 설정 열거형(굴림체·돋움체·맑은고딕 = 공문 타이포 사실상 전부)으로 목적지 글꼴을 선언받아 해결 — 원문 렌더가 선언을 따르고(넘기려는 글꼴로 미리 확인 = 그대로 넘어가는 감각), 린트는 선언-조건부(비례폭 선언에서만 연속 공백 정렬 경보 + 전각 치환 제안). 클립보드는 평문(문자·공백만) 유지.
  • F37 재처분: 갭 실재·처방 교체(배치 파일 → 루프 마찰 제거). 일괄 저장은 수요 실증 시 재개 가능한 불채택. 라인구분 vs 분할·일괄 빈칸 경보 미결 동반 소멸.

1잔여 미결

  • 대규모 색인 압축: 수백 건 큐에서 점 색인의 문법(구간 묶음·스크러버 등) — 구현 라운드 소관, 블록 1 규모 문법 재사용 후보.
  • 기존 건별 「저장」 강등 여부: 산출물 무소유 논리가 동일 적용되나, 있는 기능의 제거는 별도 수요 확인 필요(삭제는 의무를 상속한다).
  • 미루기 동사 채택: 미처리 카드를 큐 뒤로 보내는 보류 동사(재배치안에서 파생) — 이 데모는 미포함, 필요성 토의.
  • 재배열 전례 긴장의 기록: 처리분 후미 이동은 사용자-원인·색인 점 이동이라 배열 안정 전례(대표값 재배열 폐기)와 구분되나, 합의문에 구분 근거를 명기할지.
  • 글꼴 동승 재개 조건: 다중 포맷 클립보드(평문+리치)는 기술적으로 가능하나 U+00A0 오염(문자 충실 훼손)·리치 붙여넣기 정규화 편차·목적지 미확인의 세 위험으로 유보 — 기안작성기 입력 특성(평문/리치) 실증 후에만 재개, 접합부만 기록.
