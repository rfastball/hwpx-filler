칩-라이브 계약 데모

계약 한 줄: 미확정·미접촉 행의 제안은 활성 헤더를 따라 라이브로 다시 선다 — 수동 지정·확정 행은 사람 소유라 칩이 못 덮는다(그 소스 헤더를 끄면 조용히가 아니라 시끄럽게 강등된다). 아래는 실제로 동작합니다 — 칩과 표를 눌러보세요.

사용할 헤더
미사용 헤더는 자동 제안과 소스 후보에서 빠집니다. 칩을 클릭하면 다시 사용합니다.
사용 헤더를 조정하거나 표를 편집해 보세요.
템플릿 필드데이터항목미리보기 (1행 표본)상태
데이터 미리보기
이걸 눌러보세요
  1. 「품명」 칩을 끈다 → {{품명}} 제안이 '세부품명'으로 즉시 다시 선다(라이브 재제안). 다시 켜면 '품명'으로 복귀.
  2. {{공고번호}}를 「확정」하고 1번을 반복 → 확정 행은 미동도 없다(사람 소유). 이제 「공고번호」 칩을 끄면 — 시끄러운 강등.
  3. 아무 행의 데이터항목을 직접 골라본다(수동) → 칩을 아무리 토글해도 그 행은 안 바뀐다. 드롭다운 맨 아래 「(자동 제안으로 되돌리기)」로 다시 시스템 소유.
  4. 「전체 미사용」 → 원하는 칩만 켜 간다(고른다→매핑한다). 확정 행이 있으면 전체 미사용이 거부되는 것도 확인.
  5. 칩을 끄면 데이터 미리보기의 그 열이 뷰에서 즉시 빠진다(F21 라이브 압축) — 다시 켜면 복귀. 매핑 표의 행은 반대로 안 숨는다: 후보 잃은 필드는 제자리에서 노랗게 남는 것이 의무(조용한 빈칸 금지).

상태: 제안=시스템 소유(노랑, 칩 따라 유동) · 수동=사람 지정, 미확정 · 확정=사람 소유, 불가침 · 후보 없음=활성 헤더에 맞는 후보 부재. 실토큰 사본 · R-flow 토의용 소형 목업(2026-07-19) — docs/r-flow-mockups/chip-live-demo.html
