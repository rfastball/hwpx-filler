## 0. 목적과 판정 틀

R-info 13조항이 레일 「실행」 제거를 확정하며 조건을 달았다: 구현은 「작업」 패널 착지와
동시 + **계약 승계 점검 의무** — "삭제는 의무를 상속한다"(F40 조용한 드롭 전례 방지).
이 점검표는 소멸하는 표면(`screen_run.py`+`web/js/screens/run.js`, txt 적출분)에 쌓인 계약
전수를 새 거처에 결속하거나 사망 사유를 적는다.

**구조 관찰(토의 확정)**: 계약 대부분은 링1(`run_state.py`·`selection_state.py`·
`screens.py` 공유 관문)이 소유하고, 죽는 것은 링2 표면뿐이다. 상태 어휘:

| 상태 | 뜻 |
|---|---|
| **자동승계** | 링1 재사용으로 계약이 구조적으로 생존 — 구현은 배선·소비 확인만 |
| **표면 재구현** | 링2 소멸분 — 확정 설계(D1·D2 등)에 따라 패널에서 재구현 |
| **신설** | 블록 1~5가 만든 새 계약(승계가 아니라 탄생 — 참조용 수록) |
| **잔존(미노출)** | seam 존치·표면 사망 — 조용한 드롭이 아님을 명문화 |
| **분기** | 상위 미결에 묶여 두 갈래를 선기록, 라운드 종결에서 닫음 |
| **사망** | 대체물 명시와 함께 소멸 |

## 5. 파생 구현 의무(구현 라운드 착수 시 이슈 발행)

1. **#25 갱신**: 네이티브 confirm 이관 목록이 스테일(3곳) — 실물 전수 **confirm 15곳 + prompt 1곳**(editor 5·home 5+prompt 1·template 3·pool 2·relink 1·run 1)으로 갱신 등재, modal.js 이관. `window.alert` 는 통지 성격(Enter-반사 파괴 위험 없음)이라 **별도 후속 검토로 분리**(D3④).
2. **재유입 방지 가드**: web/js 에서 `window.confirm`·`window.prompt` 검출 금지 테스트(R-copy 가드 계열).
3. **아키텍처 가드**: 패널 컨트롤러가 `RunViewModel`/`SelectionModel` 을 임포트(재구현 금지)한다.
4. **식별 요약 링1 함수 + 4장면 회귀 테스트**(나라 53열·합성 함정·토큰 모드·진성 중복 백스톱 — 시연 v1.2 부록이 케이스 정본).
5. **docstring 스코프 이관**(§4-33) — 리뷰 체크.
6. 블록 4 기존 의무 재확인: T1~T5 가드·직전 필터 재적용·selection.js 중앙화 표면 전수·프루닝 의미론.
