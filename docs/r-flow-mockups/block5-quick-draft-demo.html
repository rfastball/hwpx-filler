휘발 쌍둥이: 아무것도 저장하지 않는 작업

빠른 기안을 작업의 휘발 쌍둥이로 재정의한 합의의 첫 실물입니다. 두 방향 (라이브러리×수기 / 붙여넣기×데이터)과 표현형 3층(자동 추측 → 드롭다운 정정 → 직접 수정), 선언 없는 휘발 매핑(정확=자동·근사=제안), 휘발도 가드, 승격 2동사(엄격성 국경)가 전부 동작합니다.

1. 걸어보기: 두 방향, 한 표면

권장 동선 — ① 방향 B 버튼으로 붙여넣기×데이터 상태를 만들고 개찰일시·추정가격의 자동 서식과 표시형 드롭다운 정정, 빨강 사업명칭의 근사 제안 원클릭을 확인. ② 값 하나를 직접 고쳐 「직접 수정」 강등을 본 뒤 행을 바꿔 재겨눔 가드를 확인. ③ 「복사」 후 「새 기안」은 조용히 통과하고, 복사 없이 「새 기안」을 누르면 가드가 선다. ④ 「작업으로 저장」에서 미확정 매핑 초안(엄격성 국경)을 확인.

㉠휘발 쌍둥이 — 작업과 같은 개체 구조 (템플릿×레코드 결합)에서 영속성만 0. 사분면 3칸(반대각 2+양손 기존 1회 결합)을 한 표면이 받고, 양손 즉석은 막지 않되 길을 내지 않는다(결합을 묻는 문법이 자연 차단). 경계는 기능이 아니라 반복이냐 1회냐. 슬롯은 동시 1개, 새 기안은 슬롯 덮어쓰기.
㉡표현형 3층 — 결속 순간 최선 추측을 그냥 적용하고(보이는 추측), 드롭다운으로 정정하고(프리셋 키=매핑 fmt와 1:1), 직접 수정은 사람 소유 강등(최후 수단). 어느 층이든 결과는 평문 문자열로 폼에 착지 — D-6(인라인 포매터 폐기)과 render_record는 건드리지 않는다. 사람이 표현형 층이고, 기계는 초안을 깐다.
㉢완화 조항과 그 경계 — 뷰가 곧 산출물이고 사람이 그것을 응시하며 복사하므로 "조용한"이 성립할 공간이 없다. 그래서 틀리면 보이는 추측(표현형)은 묻지 않고 적용한다. 반면 틀려도 그럴싸한 부류(열 오결속: 계약금액이 추정금액 자리에 들어가도 눈이 못 잡는다)는 엄격 유지 — 근사 일치는 자동 결속 금지, 보이는 제안 원클릭까지만.
㉣선언 없는 휘발 매핑 — 제스처는 값을 집고 선언은 관계를 집는데, 단건에서는 값=관계가 붕괴해 제스처로 충분하다(규모 상한은 정의에서 연역). 소스 드롭다운은 실물 열 목록에서만 고른다(자유 타이핑 없음, UI_DESIGN_DECISIONS 승계). 행마다 컨트롤을 상시 노출할 수 있는 것도 같은 규모 논거 — 팝오버 간접층이 불필요해졌다(시연 발견 ⓑ).
㉤행 재겨눔 3분류 — 결속(자동)은 관계라서 새 행 값으로 조용히 재생성. 직접 수정 필드는 사람 노동이면서 행 대표값이라(소실+혼합 레코드 이중 위험) 가드 후 자동 서식으로 복귀. 무결속 수기 값은 행 독립 내용이라 유지+조용한 고지 한 줄. 혼합 레코드 오염은 이 분류가 상류에서 차단한다.
㉥휘발도 사다리 — 레일 이탈·복귀=생존 (블록 4 필터 정의와 같은 칸) · 새 기안=가드+덮어쓰기 · 앱 종료=무가드 소멸(T5 수용 동형) · 승격=동결. F11("확인 없이 초기화")은 수기 폼 신설로 전제가 깨져 부분 개정 — 미소비 수기 입력·직접 수정이 있으면 수치 재진술 가드, 복사가 무장을 해제한다.
㉦승격 = 엄격성 국경 — 「작업으로 저장」은 세션의 결속·표시형 키를 매핑 초안으로 1:1 동결하되 전 행 미확정으로 착지시킨다. 고반복·불투시 지대로 넘어가는 순간 에디터의 확정 워크플로가 엄격 체제를 재부과 — 완화는 지속성 계급에 스코프되고 빚은 국경에서 갚는다. 급조 버퍼는 템플릿 저장을 내포한다(F41 폴더 묻기).
㉧빈칸 = 사후 경보 승계 — 미채움 토큰은 미리보기에 빨강 {{토큰}}으로 정직 노출되고, 복사를 막지 않되 복사 직후 수치 재진술로 시끄럽게 알린다(현행 announce 승계). 데이터 해제 시 결속 값은 평문으로 동결되어 사람 소유가 된다(시연 발견 ⓐ — 평문 착지 기층의 배당).
㉨누름틀 채움 표지(v1.1, 사용자 소급 제안) — 치환된 값에 앱 렌더 전용 음영 표지를 얹어 원문(리터럴)/ 채움(음영)/미채움(빨강) 삼분을 완성한다. 한글 누름틀의 화면 음영과 동형: 화면에서는 구분되고 클립보드에는 흐르지 않는다(캡션으로 명기해 "이대로 복사됩니다"의 정직성 유지). 소급 범위 = 표준 텍스트 기안 경로 전체(패널 기안 세션 미리보기·블록 3 큐 원문 렌더) — 라운드 종결 시 블록 3 문안에 합류.
㉩템플릿 = 세션 사본, 라이브 편집(v1.4) — 라이브러리 유래도 참조가 아니라 사본이라 두 출발 모두 원문 편집 탭에서 자유 수정, 타이핑이 토큰 폼을 즉시 재구성한다. 편집 순간 라벨 정직 강등 「(수정됨)」(칩-라이브 소유권 강등 동형). 역반영 자동 없음 — 명시 승격 「템플릿으로 저장」만, 동명 덮어쓰기는 확인 게이트. 정규 기안 세션과의 비대칭은 원리다: 안정화된 레시피의 소비 표면(편집= 에디터 소관) vs 안정화 이전의 임시 작업(편집·소비 미분화가 본질) — 지속성 축 비대칭 3호(제스처 매핑·완화 조항에 이어).
㉪데이터 소스 이원(v1.4) — 등록 데이터(자산, 읽기 전용 참조)와 임의 파일(무등록 임시 겨눔) 둘 다 받는다. 파일을 물려도 데이터 관리에 등록되지 않는다 — 라이브러리 비오염 정책의 데이터 판. 표 행 클립보드 붙여넣기는 미결, '겨눔' 어휘 정정은 R-copy 소관.

2. 이 시연이 묻는 것

블록 5 제안: 빠른 기안 = 휘발성 작업. 존치 확정 위에서 본질을 재정의하고, 표현형은 자동 추측+드롭다운 정정 +직접 수정의 3층으로, 매핑은 선언 없는 제스처 결속으로, confirm-or-alarm은 가시성 경계까지 완화하되 승격 국경에서 재확정한다.

?확정 요청 5

  • 표면 형상 — 슬롯 행(템플릿·데이터) + 값 폼·미리보기 2판 + 승격 동사 푸터. 인라인 소스·표시형 드롭다운이 팝오버를 대체한 형상(시연 발견 ⓑ)이 성립하는가.
  • 행 재겨눔 3분류(㉤) — 결속=조용 재생성 · 직접 수정=가드 · 수기=유지가 혼합 레코드 차단으로 충분한가.
  • 승격 게이트 형상(㉦) — 작업 저장이 템플릿 저장을 내포하는 1게이트 (이름 2개를 한 모달에서), 매핑 초안 전 행 미확정 착지.
  • 휘발도 사다리 잔여 2(㉥) — 앱 종료 무가드(T5 동형)와 슬롯 동시 1개.
  • 누름틀 표지 소급 범위(㉨) — 이 표면을 넘어 패널 기안 세션·블록 3 큐 원문 렌더까지 삼분 표지(원문/채움/미채움)를 적용하는 결정.

＋증보 판단 3 (v1.4, 승인 직후 재개)

  • 데이터 소스 이원(㉪) — 등록 데이터+임의 파일(무등록), 비오염 정책의 데이터 판 명문화.
  • 템플릿 세션 사본+라이브 편집(㉩) — 두 출발 모두 원문 편집, (수정됨) 정직 강등, 역반영은 명시 승격만(동명 덮어쓰기 확인 게이트).
  • 비대칭의 원리 — 정규 기안 세션은 템플릿 수정 불가(안정화된 레시피), 빠른 기안은 편집·소비 미분화(안정화 이전) — 지속성 축 비대칭 3호로 합의문 기록.

…미결 (구현 라운드 또는 후속 토의)

  • 템플릿 전환 가드 — 작성 중 템플릿을 바꾸면 동명 토큰 값은 승계하고 나머지는 버리는데, 이 전이에 가드가 필요한가(시연은 무가드 승계).
  • 표지 변주 — 누름틀 음영의 토글 제공 여부(한글 동형)와 소유권별 색 분화 (자동/직접) 여부. 시연은 단일 음영+호버 툴팁(토큰명·상태)으로 절제.
  • 근사 판정의 실구현 — 시연은 접두 일치 휴리스틱. 실구현은 블록 4 자모 분해 유틸 재사용 검토.
  • 붙여넣기의 라이브 승격 — 모달 확정식 유지 vs 인라인 편집(실시간성 확장). 현행 modal.js 관례와의 트레이드오프.
  • 글꼴 선언·정렬 린트 합류 — 블록 3 전역 값 승계 확인(시연은 드롭다운만 배선).
  • 레일 문구 — 「빠른 기안」 레일 설명문(R-copy 소관)과 빈 상태 카드 문안.

작업으로 저장

이 결합을 반복할 수 있게 작업으로 승격합니다. 세션의 결속과 표시형 선택이 매핑 초안으로 동결됩니다.

매핑 초안은 전 행 미확정으로 저장됩니다. 실행 전에 작업 에디터에서 확정해야 합니다.
