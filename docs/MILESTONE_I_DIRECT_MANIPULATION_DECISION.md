# Milestone I — 직접 조작 스코프 결정 (#222)

기록일: 2026-07-23

| 후보 | 결정 | 근거·후속 |
|---|---|---|
| 파일 드래그-드롭 등록 | 채택 | 드롭은 겨눔 제안으로만 쓰고 기존 확장자·시트·덮어쓰기 게이트에 합류한다. 구현은 [#274](https://github.com/rfastball/hwpx-filler/issues/274). |
| 필터 테이블 드래그 범위 선택 | 채택 | 기존 `select_range`가 판정을 계속 소유하므로 정확성 계약을 우회하지 않는다. 구현은 [#275](https://github.com/rfastball/hwpx-filler/issues/275). |
| 마스터 목록 폭 스플리터 | 채택 | 순수 뷰 기하이며 긴 이름 말줄임을 사용자가 직접 완화할 수 있다. 구현은 기존 [#221 S7](https://github.com/rfastball/hwpx-filler/issues/221). |
| 작업 카드→그룹 드래그 | 기각 | 현재 이동 다이얼로그가 대상 그룹과 병합/이동 결과를 재진술한다. 직접 드롭은 이 가드를 우회하고 오드롭 복구 장치도 없어 프로젝트의 정확성 우선 원칙과 충돌한다. |

## 경계

- 채택은 기존 판정 경로에 합류한다는 뜻이며, 드롭 자체가 새 저장/덮어쓰기 권한을 갖지 않는다.
- 파일 드롭과 범위 선택은 Milestone I의 결정-only 범위를 다시 늘리지 않도록 백로그 후속으로 둔다.
- 스플리터만 이미 Milestone I #221의 완주 항목이므로 이 결정 뒤 바로 구현한다.
- 작업 카드→그룹 드래그는 재론하지 않는다. 이동 다이얼로그와 동등한 사전 재진술 및 오드롭
  복원 모델이 먼저 제안되는 경우에만 새 결정 이슈로 재개한다.
