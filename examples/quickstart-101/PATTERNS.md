# hwpxfiller 102 — 세 가지 실전 조합 (GUI)

[README](README.md) 의 101(작업 하나 만들어 실행)을 뗐다면, 실무는 대개 이 셋이다.
전부 화면에서, 같은 `quickstart-101/` 자산만으로 재현된다. 먼저 **[`start-101.cmd`](start-101.cmd)
를 더블클릭**해 앱을 띄운다(101 준비 참고 — 런처가 이 폴더를 홈으로 잡아 준다).

1. **한 데이터 → 여러 템플릿** — 한 벌 데이터로 여러 문서를 한 번에.
2. **여러 데이터 → 한 템플릿** — 여러 배치(부서·월별 파일)를 한 템플릿으로.
3. **저장된 작업(Job) 간 전환** — 반복 업무를 골라 실행.

먼저 **작업 두 개**를 만들어 둔다(101 의 "① 작업 만들기"를 두 번). 아래 셋이 이 둘을 쓴다.

- 작업 `발주요청서` — 템플릿 `templates/발주요청서.hwpx`, 파일명 `발주요청서-{{공고번호}}`.
- 작업 `구매요청서` — 템플릿 `templates/구매요청서.hwpx`, 파일명 `구매요청서-{{공고번호}}`.

> 팁: **템플릿 관리** 화면에는 이 폴더의 두 템플릿이 이미 떠 있다. 카드의
> **[작업 만들기]** 를 누르면 그 템플릿이 채워진 편집기로 바로 들어간다.

---

## 1. 한 데이터 → 여러 템플릿

**실행 화면에서 작업만 바꿔 반복 실행**한다 — 실행은 한 종류뿐이고, 같은 데이터를
작업별로 한 번씩 돌리면 된다.

1. 대시보드 → `발주요청서` 카드 **[실행]** → **데이터**: **[파일 선택…]** →
   `data/발주목록.csv` → 저장 폴더 지정(예: `저장폴더/발주요청서`) → **[문서 생성]**. (3건)
2. **같은 실행 화면에서** 상단 **작업 선택**을 `구매요청서` 로 바꾼다 → 같은
   `data/발주목록.csv` 를 다시 선택 → 저장 폴더(예: `저장폴더/구매요청서`) →
   **[문서 생성]**. (3건)

```
저장폴더/
├─ 발주요청서/  발주요청서-2026-001.hwpx  …-002  …-003
└─ 구매요청서/  구매요청서-2026-001.hwpx  …-002  …-003
```

**부분집합 템플릿은 필요한 필드만 쓴다.** `구매요청서` 는 4필드(수요기관·품명·추정가격·
담당자)만 채우므로 데이터의 남는 열은 그냥 안 들어간다 — 값이 사라진 게 아니라 이
템플릿이 안 쓸 뿐이다.

> 자주 쓰는 데이터라면 **데이터 관리**에 등록해 두거나 작업의 **선언 데이터 자동
> 등록**을 쓰면, 작업을 바꿀 때마다 파일을 다시 고르는 대신 자동으로 연결된다.

> 텍스트 초안까지 한 벌에 넣고 싶으면, HWPX 는 위처럼 일괄 생성하고 기안문은 **즉시
> 기안** 화면에서 같은 `발주목록.csv` 로 뽑는다(README 트랙 B). "한 데이터 → 3종 산출".

## 2. 여러 데이터 → 한 템플릿

작업 하나(`발주요청서`)를 **실행 화면에서 데이터만 바꿔 반복 실행**한다. 같은 저장
폴더로 내면 결과가 **누적**된다. (부서별·월별로 파일이 쪼개져 오는 실무에 맞는다.)

1. 대시보드 → `발주요청서` 카드 **[실행]**.
2. **데이터**: **[파일 선택…]** → `data/발주목록.csv` → 레코드 확인 → 저장 폴더 지정 →
   **[문서 생성]**. (3건)
3. **같은 실행 화면에서** **데이터**만 **[파일 선택…]** → `data/발주목록_2.csv` 로 바꾼다
   → **[문서 생성]**. (2건, **같은 저장 폴더**)

저장 폴더에 3 + 2 = 5개가 누적된다(`…-001~003` 다음 `…-101, …-102`).

**같은 파일명이 겹치면 두 번째 생성에서 "덮어쓰기 확인"이 뜬다.** 두 배치의
`공고번호` 가 겹쳐 파일명이 같아지면, 앞서 만든(손보정했을 수 있는) 산출물을 말없이
덮지 않으려고 대화상자로 막는다. 정말 교체할 거면 **[덮어쓰고 진행]**. (이 예제는
번호가 안 겹쳐 그냥 누적된다.)

> **자주 쓰는 데이터라면 데이터 풀에 등록.** 레일 **데이터 풀** 에 두 파일을 참조로
> 등록해 두면, 실행 화면에서 **[데이터 풀에서…]** 로 매번 파일 찾기 없이 고를 수 있다
> (경로만 저장하고 실행 때 다시 읽는다 — 스냅샷 아님).
>
> **정말 "합쳐서" 한 표로?** 여러 소스를 키로 조인(merge)하거나 이어붙여(append) **한
> 데이터로 만드는** 조립은 데이터 풀의 파이프라인(Power-Query 식) 기능이다. "행을 그냥
> 이어 문서를 쭉 뽑는" 수준이면 위처럼 **반복 실행**이 가장 단순하다.

## 3. 저장된 작업(Job) 간 전환

앞의 것들이 보여주듯, 한 번 저장한 작업은 템플릿·매핑·파일명을 **기억**한다. 매번 다시
설정하지 않고 **골라 실행**하는 게 핵심이다.

**실행 슬롯은 하나다.** 어느 작업을 그 슬롯에 얹느냐가 곧 "전환"이다.

1. 대시보드에 카드가 여럿 있다(`발주요청서`, `구매요청서`, …).
2. `발주요청서` 카드 **[실행]** → 실행 화면이 그 작업으로 열리고 레일에
   **"실행: 발주요청서"** 가 뜬다. 데이터 물려 생성.
3. 레일의 **대시보드** 로 돌아가 이번엔 `구매요청서` 카드 **[실행]** → 실행 슬롯이 그
   작업으로 **교체**되고 레일이 **"실행: 구매요청서"** 로 바뀐다.
   (직전 실행이 진행 중이면 **"생성 중단"** 을 먼저 확인한다 — 조용히 버리지 않는다.)

즉 전환 = "대시보드로 → 다른 카드의 **[실행]**". 템플릿·패턴을 다시 타지 않는다.

**내용 바꾸기·지우기**

- 카드 **[작업 편집]** → 저장된 값이 채워진 편집기가 열린다(템플릿·매핑·파일명 수정 후
  **[작업 저장]**).
- 카드 **[삭제]** → 확인 후 제거(기본 선택은 취소).

**저장된 작업의 실제 모습.** 작업은 `jobs/발주요청서.job.json`(이 폴더 밑, 홈 지정 덕에)
으로 남는다. 열어 보면 `{템플릿 + 매핑 + 파일명}` + 약간의 메타뿐 — **데이터·행은 없다.**

```json
{
  "version": 1,
  "name": "발주요청서",
  "template_path": ".../templates/발주요청서.hwpx",
  "filename_pattern": "발주요청서-{{공고번호}}",
  "mapping": { "name": "발주요청서", "mappings": [ /* 필드↔소스 */ ] },
  "last_run_at": "2026-07-14T..."
}
```

> **더 나아가기.** 자주 쓰는 매핑을 **매핑 프로파일**(베이스 매핑)로 한 번 선언해 여러
> 작업이 물려받게 하거나, 작업이 많아지면 대시보드에서 **분류 태그**로 묶어 보는
> 그룹·필터 렌즈를 쓸 수 있다(작업 저장 4단계의 "분류 태그"에서 붙인다).
