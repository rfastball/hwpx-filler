# 문서나르미 102 — 세 가지 실전 조합 (GUI)

[README](README.md) 의 101(작업 하나 만들어 문서 생성)을 뗐다면, 실무는 대개 이 셋이다.
전부 화면에서, 같은 `quickstart-101/` 자산만으로 재현된다. 먼저 **[`start-101.cmd`](start-101.cmd)
를 더블클릭**해 앱을 띄운다(101 준비 참고 — 런처가 이 폴더를 홈으로 잡아 준다).

1. **한 데이터 → 여러 템플릿** — 한 벌 데이터로 여러 문서를 한 번에.
2. **여러 데이터 → 한 템플릿** — 여러 배치(부서·월별 파일)를 한 템플릿으로.
3. **저장된 작업(Job) 간 전환** — 반복 업무를 골라 쓰기.

먼저 **작업 두 개**를 만들어 둔다(101 의 "① 작업 만들기"를 두 번). 아래 셋이 이 둘을 쓴다.

- 작업 `발주요청서` — 템플릿 `templates/발주요청서.hwpx`, 파일명 `발주요청서-{{공고번호}}`.
- 작업 `구매요청서` — 템플릿 `templates/구매요청서.hwpx`, 파일명 `구매요청서-{{공고번호}}`.
  (2단계에서 같은 `data/발주목록.csv` 를 고르면 4필드가 똑같이 자동 제안된다.)

> 팁: **템플릿 관리** 화면에도 이 폴더의 두 템플릿이 떠 있다 — 카드에서 **작업 만들기**로
> 그 템플릿이 채워진 편집 모드로 바로 들어갈 수 있다.

---

## 1. 한 데이터 → 여러 템플릿

**작업만 바꿔 같은 데이터로 한 번씩 생성**한다.

1. 왼쪽 목록에서 `발주요청서` 선택 → (101 대로면 기본 데이터가 자동 연결돼 있다) →
   **[이 작업으로 문서 생성]**. (3건)
2. 목록에서 `구매요청서` 선택 → **② 데이터 연결**에서 같은 `data/발주목록.csv` 를
   겨눈다(**[파일 선택…]**, 또는 등록해 뒀다면 **[등록 데이터…]**) → **[이 작업으로
   문서 생성]**. (3건)

기본 저장 폴더(`templates\Results`)에 여섯 개가 나란히 생긴다 — 파일명 앞머리가
달라(`발주요청서-…` / `구매요청서-…`) 겹치지 않는다.

**부분집합 템플릿은 필요한 필드만 쓴다.** `구매요청서` 는 4필드(수요기관·품명·추정가격·
담당자)만 채우므로 데이터의 남는 열은 그냥 안 들어간다 — 값이 사라진 게 아니라 이
템플릿이 안 쓸 뿐이다.

> 텍스트 초안까지 한 벌에 넣고 싶으면, HWPX 는 위처럼 일괄 생성하고 기안문은 **기안**
> 화면에서 같은 `발주목록.csv` 로 뽑는다(README 트랙 B). "한 데이터 → 3종 산출".

---

## 2. 여러 데이터 → 한 템플릿

작업 하나(`발주요청서`)에 **데이터만 바꿔 다시 생성**한다. 같은 저장 폴더로 내면
결과가 **누적**된다. (부서별·월별로 파일이 쪼개져 오는 실무에 맞는다.)

1. 목록에서 `발주요청서` 선택 → 기본 데이터(발주목록)로 **[이 작업으로 문서 생성]**. (3건)
2. **② 데이터 연결**의 **[파일 선택…]** → `data/발주목록_2.csv` 로 바꾼다 →
   **[이 작업으로 문서 생성]**. (2건, **같은 저장 폴더**)

저장 폴더에 3 + 2 = 5개가 누적된다(`…-001~003` 다음 `…-101, …-102`).

**같은 파일명이 겹치면 두 번째 생성에서 "덮어쓰기 확인"이 뜬다.** 두 배치의
`공고번호` 가 겹쳐 파일명이 같아지면, 앞서 만든(손보정했을 수 있는) 산출물을 말없이
덮지 않으려고 대화상자로 막는다. 정말 교체할 거면 **[덮어쓰고 생성]**. (이 예제는
번호가 안 겹쳐 그냥 누적된다.)

> **자주 쓰는 데이터라면 데이터 관리에 등록.** 레일 **데이터 관리**에 파일을 참조로
> 등록해 두면, 작업·기안 화면의 **[등록 데이터…]** 로 매번 파일 찾기 없이 고를 수 있다
> (경로만 저장하고 생성 때 다시 읽는다 — 스냅샷 아님). 101 의 "데이터 함께 등록"이
> 바로 이 등록을 저장길에 겸사로 해 준 것이다.

## 3. 저장된 작업(Job) 간 전환

앞의 것들이 보여주듯, 한 번 저장한 작업은 템플릿·매핑·파일명을 **기억**한다. 매번 다시
설정하지 않고 **골라 쓰는** 게 핵심이다.

**세션 슬롯은 하나다.** 왼쪽 목록에서 어느 작업을 클릭하느냐가 곧 "전환"이다 —
오른쪽 세션 패널이 그 작업의 템플릿·기본 데이터·저장 폴더로 통째로 갈아입는다.
(진행 중이던 세션 상태를 잃을 수 있는 전환이면 앱이 먼저 확인을 띄운다 — 조용히
버리지 않는다.)

**내용 바꾸기·지우기** — 목록 행에 마우스를 올리면 나오는 **⋮ 메뉴**:

- **편집** → 저장된 값이 채워진 편집 탭(템플릿·매핑·파일명 수정 후 **[저장]**).
- **이름 변경** → 행이 입력칸으로 바뀐다(Enter 확정 · Escape 취소).
- **그룹으로 이동…** → 작업이 많아지면 그룹 구획으로 묶는다.
- **삭제** → 확인 후 제거.

**저장된 작업의 실제 모습.** 작업은 `jobs/발주요청서.job.json`(이 폴더 밑, 홈 지정 덕에)
으로 남는다. 열어 보면 `{템플릿 + 매핑 + 파일명}` + 약간의 메타뿐 — **데이터·행은 없다.**

```json
{
  "version": 1,
  "name": "발주요청서",
  "template_path": ".../templates/발주요청서.hwpx",
  "filename_pattern": "발주요청서-{{공고번호}}",
  "mapping": { "name": "발주요청서", "mappings": [ /* 필드↔소스 */ ] }
}
```
