# nara_mulpum — 실 나라장터 물품 공고 탐색 테스트셋

`tests/corpus/scenario/`(작은·정제·결정적 골든)와 **역할이 다르다.** 이 폴더는
조달 실무 워크플로를 **실데이터의 지저분함**으로 두들기기 위한 탐색용 코퍼스다.
수동 테스트 세션(session-based / Whittaker tour)이 여기서 입력을 가져온다.

- **형식**: `response.body.items[]` 봉투 — `NaraStdDataSource.parse()` 가 그대로 소비
  (얼릴 때 parse 왕복 검증 통과). 라이브 호출 없이 결정적으로 리플레이된다.
- **PII 마스킹**(공개 데이터이나 공무원 개인정보는 중립화):
  - `*OfclNm`(담당자명) → `담당자`
  - `*Tel`(전화) → 숫자 전부 `0`(형식·길이 보존)
  - `*EmailAdrs`(이메일) → `official@example.go.kr`
  - 기관명·기관코드·공고명·금액·일시는 **원문 유지**(공개 정보, 테스트 신뢰성).

## 첫 취득 패스 findings — 실행 검증 완료(2026-07-14)

수확 자체가 첫 테스트 패스였다. 아래 4건을 실 코드(mapping/engine/batch/naming)에
이 세트로 태워 **적대적으로 반증 시도**했다. 결과: 3건 반증(앱이 confirm-or-alarm 준수),
1건 확정하되 버그 아닌 마찰로 강등. verify 재현은 `scripts/`(수확기 옆) 참고.

- **F1 — 날짜 필드 통째 결측** · 표적 R26BK01621756 → **반증(정상)**. 빈 날짜는
  `format_engine`이 `''`로 degrade하고 엔진이 빈값을 스킵(`engine.py:42`) → 누름틀이
  산출문서에 **잔존(loud)**. 크래시·깨진 출력 없음.
- **F2 — 담당부서 매핑되나 소스 0/50 빈값** → **반증(정상), 잔여 관찰 有**. `''`는
  엔진 active에서 제외 → 누름틀 잔존(조용한 채움 아님). **단** 배치 리포트
  (`applied`/`unmatched`)엔 "기대했으나 빈값" 신호가 없다 — 문서를 눈으로 안 보면
  raw 누름틀 채로 나갈 여지(설계 질문, 심각도 낮음).
- **F3 — 공고명 중복 → 파일명 충돌** → **반증(정상)**. 동명 Adobe 2건 배치 → 파일
  **2개**(`OutputNamer`가 `_1` 접미사로 유일화), `succeeded=2`, 손실 0. 디스크 기존
  파일과 충돌 시 `OutputCollisionError`로 착수 전 차단(덮어쓰기 확정 요구). 손실 방어 견고.
- **F4 — 라벨 사전 노후** → **확정, 단 강등(마찰)**. 실 서비스 53키 중 `_FIELD_LABELS`
  미수록 18키: `elctrnBidYn`(전자입찰)·`refNtceNo/Ord`(참조공고)·`rsrvtnPrceDcsnMthdNm`
  (예가방법)·`intrntnlBidYn`·`ppsNtceYn`·`cmmn*`(공동수급)·`presnatnOprtn*`·
  `dataBssDate`·`*EmailAdrs`·`dmndInsttCd` 등. 미라벨 키는 `suggest_mappings` 퍼지
  초안에 안 걸려 수동 매핑 필요 — 손상 아닌 커버리지 갭. **조치: `_FIELD_LABELS` 갱신
  (GitHub 이슈).**

**승격 후보**(회귀로 잠글 좋은 동작): F1(빈 날짜 무크래시·누름틀 잔존)·F3(동명→유일
파일명·무손실)은 실 세트 대비 회귀 테스트로 고정할 가치가 있다.

## 재수확

`.secrets/nara_service_key` 로 다시 긁으려면 `python scripts/build_nara_testset.py`
(`--dry-run` 으로 미리보기). **라이브 취득이라 비결정적** — 재실행은 그 시점 공고로 새
세트를 만든다(이 `mulpum.json` 이 동결본). 취득 창은 최대 1개월. 갱신 시 위 매니페스트도 손본다.
